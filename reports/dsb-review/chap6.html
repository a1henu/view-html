第六章 排序

排序是计算机科学最基本的操作之一，掌握各种排序算法的原理、复杂度和适用场景是数据结构课程的核心内容

6.1排序的基本概念

定义排序 (Sorting)

将一组数据元素（记录）按照某个关键字的值递增（或递减）重新排列的过程。

定义排序的稳定性

若待排序的序列中存在两个关键字相等的记录 $R_i$ 和 $R_j$（$i < j$），排序后 $R_i$ 仍然在 $R_j$ 前面，则称该排序算法是稳定的；否则称为不稳定的。

注意：稳定性是排序算法本身的属性，与具体输入无关。只要存在一个反例使相等元素顺序改变，就称该算法不稳定。

定义内部排序 vs 外部排序

内部排序：待排序的所有记录全部存放在内存中进行排序。

外部排序：待排序的记录数量很大，内存无法容纳全部记录，排序过程中需要对外存进行多次访问。

洞察排序算法的评价指标
  • 时间复杂度：比较次数 + 移动（交换）次数
  • 空间复杂度：除原始数据占用的空间外，算法还需要的辅助存储空间
  • 稳定性：对于多关键字排序和应用场景很重要

基于比较的排序算法，时间复杂度下界为 $\Omega(n \log n)$。

6.2冒泡排序 (Bubble Sort)

定义冒泡排序原理

反复遍历待排序序列，每次比较相邻的两个元素，如果它们的顺序错误就交换它们。每一趟遍历结束后，当前未排序部分中最大（或最小）的元素会"冒泡"到序列的一端。

如果某一趟没有发生任何交换，说明序列已经有序，可以提前终止。

性质冒泡排序复杂度分析
  • 最好时间：$O(n)$ —— 序列已有序，只需一趟扫描无交换即可结束
  • 最坏时间：$O(n^2)$ —— 序列逆序，需要 $n-1$ 趟，第 $i$ 趟比较 $n-i$ 次
  • 平均时间：$O(n^2)$
  • 空间复杂度：$O(1)$（只需要一个临时变量用于交换）
  • 稳定性：稳定（相等元素不交换）
例题冒泡排序手工模拟

对序列 {5, 3, 8, 1, 2} 进行升序冒泡排序（大的往右冒）：

  1. 第1趟：比较相邻元素，将最大值冒泡到末尾
    (5,3)→交换→3,5,8,1,2
    (5,8)→不换→3,5,8,1,2
    (8,1)→交换→3,5,1,8,2
    (8,2)→交换→3,5,1,2,8
    结果：{3, 5, 1, 2, 8}，最大值8归位
  2. 第2趟：对前4个元素冒泡
    (3,5)→不换; (5,1)→交换→3,1,5,2; (5,2)→交换→3,1,2,5
    结果：{3, 1, 2, 5, 8}
  3. 第3趟：对前3个元素冒泡
    (3,1)→交换→1,3,2; (3,2)→交换→1,2,3
    结果：{1, 2, 3, 5, 8}
  4. 第4趟：对前2个元素冒泡
    (1,2)→不换，本趟无交换，提前结束
    最终结果：{1, 2, 3, 5, 8}
易错冒泡排序的方向

冒泡排序可以从左往右冒（大的往右，每趟确定末尾最大值），也可以从右往左冒（小的往左，每趟确定开头最小值）。做题时要注意题目的具体描述。

关键规律：第 $k$ 趟结束后，一定有 $k$ 个元素到达了最终位置。

6.3选择排序 (Selection Sort)

定义直接选择排序原理

每趟从未排序部分中选择最小（或最大）的元素，将其与未排序部分的第一个元素交换位置，从而将其"归入"已排序部分的末尾。

经过 $n-1$ 趟选择即可完成整个排序。

性质选择排序复杂度分析
  • 比较次数：始终为 $\displaystyle\sum_{i=1}^{n-1}(n-i) = \frac{n(n-1)}{2}$，与输入无关
  • 最好 = 最坏 = 平均时间：$O(n^2)$
  • 空间复杂度：$O(1)$
  • 稳定性：不稳定
例题选择排序手工模拟

对序列 {64, 25, 12, 22, 11} 进行升序选择排序：

  1. 第1趟：从全部5个元素中找最小值 11，与位置0的64交换
    结果：{11, 25, 12, 22, 64}
  2. 第2趟：从后4个元素中找最小值 12，与位置1的25交换
    结果：{11, 12, 25, 22, 64}
  3. 第3趟：从后3个元素中找最小值 22，与位置2的25交换
    结果：{11, 12, 22, 25, 64}
  4. 第4趟：从后2个元素中找最小值 25，已在位置3，不换
    最终结果：{11, 12, 22, 25, 64}
易错选择排序为什么不稳定

经典反例：对序列 {5a, 5b, 3} 进行升序选择排序。

第1趟选择最小值3，与位置0的5a交换，得 {3, 5b, 5a}。此时5a跑到了5b后面，相等元素的相对顺序被改变。

根本原因：选择排序的交换操作是"跳跃式"的，可能把前面的元素直接甩到后面。

洞察从 $n^2$ 个数中找最小值

从 $n^2$ 个整数中找到最小的一个，至少需要比较 $n^2 - 1$ 次。这是因为"锦标赛"模型中，要确定冠军（最小值），其余 $n^2 - 1$ 个元素都必须至少输一次。

6.4插入排序 (Insertion Sort)

定义直接插入排序原理

将待排序序列分为已排序和未排序两部分。每次取未排序部分的第一个元素，在已排序部分中从后往前找到合适的位置插入，使已排序部分仍然保持有序。

初始时，第一个元素自成一个已排序序列。

性质插入排序复杂度分析
  • 最好时间：$O(n)$ —— 序列已经有序，每个元素只需比较1次即确认位置正确
  • 最坏时间：$O(n^2)$ —— 序列完全逆序，每个元素都要移到最前面
  • 平均时间：$O(n^2)$
  • 空间复杂度：$O(1)$
  • 稳定性：稳定（插入时遇到相等元素停止移动，放在其后面）
例题插入排序手工模拟

对序列 {5, 2, 4, 6, 1, 3} 进行升序直接插入排序：

  1. 初始已排序部分：{5}，取 2 插入
    2 < 5，5后移，2插到前面 → {2, 5}
  2. 取 4 插入：4 < 5，5后移；4 > 2，停止 → {2, 4, 5}
  3. 取 6 插入：6 > 5，不移动 → {2, 4, 5, 6}
  4. 取 1 插入：1 < 6, 6后移; 1 < 5, 5后移; 1 < 4, 4后移; 1 < 2, 2后移 → {1, 2, 4, 5, 6}
  5. 取 3 插入：3 < 6, 6后移; 3 < 5, 5后移; 3 < 4, 4后移; 3 > 2, 停止 → {1, 2, 3, 4, 5, 6}
对于已经有序的数据，插入排序是最快的（$O(n)$），甚至比快速排序更快（快排在正序时可能退化为 $O(n^2)$）。
易错插入排序 vs 快速排序在正序情况下

正序输入时：

  • 插入排序：每个元素只需1次比较 → $O(n)$
  • 快速排序（取第一个为基准）：每次划分极度不均（0和n-1），退化为 $O(n^2)$

所以正序时插入排序可能比快排更快，这是往年考过的判断题。

6.5希尔排序 (Shell Sort)

定义希尔排序原理

希尔排序是对直接插入排序的改进，也叫缩小增量排序。基本思想：

  1. 选择一个步长序列 $d_1 > d_2 > \cdots > d_k = 1$
  2. 对于每个步长 $d_i$，将序列分为 $d_i$ 个子组（相距 $d_i$ 的元素在同一组）
  3. 对每个子组分别进行直接插入排序
  4. 逐步缩小步长，直到步长为1（最后一趟就是普通插入排序，但此时序列已"基本有序"）
性质希尔排序复杂度分析
  • 时间复杂度：取决于步长序列，一般为 $O(n^{1.3})$ ~ $O(n^{1.5})$
  • 常用步长序列：$n/2, n/4, \ldots, 1$（Shell原始序列）
  • 空间复杂度：$O(1)$
  • 稳定性：不稳定（不同子组的排序可能改变相等元素的相对顺序）
例题希尔排序手工模拟

对序列 {46, 70, 56, 38, 40, 80, 60, 22}（共8个元素）进行降序希尔排序，初始步长为4：

  1. 步长 d = 4：将8个元素按间隔4分组
    位置01234567
    元素4670563840806022
    分组ABCDABCD
    组A: {46, 40} → 降序排序: {46, 40}（46>40，不换）
    组B: {70, 80} → 降序排序: {80, 70}（交换）
    组C: {56, 60} → 降序排序: {60, 56}（交换）
    组D: {38, 22} → 降序排序: {38, 22}（38>22，不换）
    第一趟结果：{46, 80, 60, 38, 40, 70, 56, 22}
  2. 后续步长：d=2, d=1，继续同样方法分组并插入排序，此处省略。
洞察希尔排序为什么比插入排序快

直接插入排序在序列"基本有序"时效率很高（接近$O(n)$）。希尔排序的前几趟使用较大步长，快速消除了大量的"逆序对"，使序列趋于"基本有序"。到最后步长为1时，已经几乎不需要移动了。

注意：虽然希尔排序每一趟调用的是插入排序，但其总体效率比直接插入排序高。

6.6快速排序 (Quick Sort)

定义快速排序原理

快速排序采用分治策略：

  1. 选基准（Pivot）：通常选择第一个元素
  2. 划分（Partition）：将序列划分为两部分——左边所有元素 ≤ pivot，右边所有元素 ≥ pivot
  3. 递归：对左右两部分分别递归进行快速排序
定义划分过程详解（双指针法）

以第一个元素为基准 pivot = a[low]：

  1. 设两个指针 $i = \text{low}$，$j = \text{high}$
  2. 从右向左扫描（$j$ 递减）：找到第一个 < pivot 的元素，将其填入 $a[i]$ 的位置
  3. 从左向右扫描（$i$ 递增）：找到第一个 > pivot 的元素，将其填入 $a[j]$ 的位置
  4. 重复步骤2-3，直到 $i = j$，将 pivot 填入 $a[i]$
性质快速排序复杂度分析
  • 最好时间：$O(n \log n)$ —— 每次划分都恰好均分
  • 最坏时间：$O(n^2)$ —— 序列已有序或逆序（每次划分极度不均）
  • 平均时间：$O(n \log n)$（实践中最快的排序算法之一）
  • 空间复杂度：$O(\log n)$~$O(n)$（递归调用栈的深度）
  • 稳定性：不稳定
例题快速排序划分过程模拟

对序列 {46, 70, 56, 38, 40, 80} 进行非递减排序，以第一个元素46为基准：

  1. 初始：pivot = 46，i = 0, j = 5
    [_, 70, 56, 38, 40, 80]（位置0空出）
  2. j从右扫描：a[5]=80 ≥ 46, j--; a[4]=40 < 46, 将40填入位置i=0
    [40, 70, 56, 38, _, 80]，i++, i=1
  3. i从左扫描：a[1]=70 > 46, 将70填入位置j=4
    [40, _, 56, 38, 70, 80]，j--, j=3
  4. j从右扫描：a[3]=38 < 46, 将38填入位置i=1
    [40, 38, 56, _, 70, 80]，i++, i=2
  5. i从左扫描：a[2]=56 > 46, 将56填入位置j=3
    [40, 38, _, 56, 70, 80]，j--, j=2
  6. 此时 i = j = 2，将pivot=46填入位置2
    结果：{40, 38, 46, 56, 70, 80}

基准46在位置2，左边 {40, 38} 都 ≤ 46，右边 {56, 70, 80} 都 ≥ 46。

易错快速排序的最坏情况

当输入序列已经有序（正序或逆序）且每次选第一个元素为基准时，快速排序退化为 $O(n^2)$。因为每次划分得到的子问题大小为 0 和 n-1。

对比：1000个元素找前5小时，快排需要全部排完，而堆排序只需建堆后取5次堆顶即可。因此找前k小/大的问题，堆排序更优。

6.7归并排序 (Merge Sort)

定义归并排序原理

采用分治法：

  1. 分（Divide）：将序列从中间一分为二
  2. 治（Conquer）：对左右两个子序列分别递归排序
  3. 合（Merge）：将两个已排序的子序列合并为一个有序序列

二路归并排序的每趟归并：将相邻的有序子序列两两合并。第1趟后子序列长度为2，第2趟后长度为4，...，第 $k$ 趟后长度为 $2^k$。

性质归并排序复杂度分析
  • 每趟归并的时间：$O(n)$（每个元素参与一次合并）
  • 趟数：$\lceil \log_2 n \rceil$
  • 总时间：$O(n \log n)$（最好 = 平均 = 最坏）
  • 空间复杂度：$O(n)$（需要与原序列等长的辅助数组）
  • 稳定性：稳定（合并时相等元素保持原有顺序）
例题二路归并排序手工模拟

对序列 {38, 27, 43, 3, 9, 82, 10} 进行升序归并排序：

  1. 第1趟归并（子序列长度1→2）：
    (38,27)→{27,38}; (43,3)→{3,43}; (9,82)→{9,82}; (10)→{10}
    结果：{27,38, 3,43, 9,82, 10}
  2. 第2趟归并（子序列长度2→4）：
    合并{27,38}和{3,43}→{3,27,38,43}; 合并{9,82}和{10}→{9,10,82}
    结果：{3,27,38,43, 9,10,82}
  3. 第3趟归并（子序列长度4→8）：
    合并{3,27,38,43}和{9,10,82}→{3,9,10,27,38,43,82}
    最终结果：{3, 9, 10, 27, 38, 43, 82}
归并排序是外部排序的首选算法。当数据量远大于内存容量时（如1GB数据，100MB内存），归并排序通过顺序访问磁盘的方式高效完成排序。
洞察归并排序的递推关系

归并排序的时间递推式：$T(n) = 2T(n/2) + O(n)$

由主定理（Master Theorem）：$a=2, b=2, f(n)=O(n)$，满足 $f(n) = \Theta(n^{\log_b a}) = \Theta(n)$，得 $T(n) = O(n \log n)$。

6.8堆排序 (Heap Sort)

定义堆的概念

最大堆（大顶堆）：完全二叉树中，每个节点的值都 ≥ 其子节点的值。根节点是最大值。

最小堆（小顶堆）：每个节点的值都 ≤ 其子节点的值。根节点是最小值。

堆用数组存储，对于下标从0开始的数组：节点 $i$ 的左子节点为 $2i+1$，右子节点为 $2i+2$，父节点为 $\lfloor(i-1)/2\rfloor$。

定义堆排序原理（升序排列用最大堆）
  1. 建堆：从最后一个非叶节点开始，自下而上调整，使整个数组满足最大堆性质
  2. 排序：重复以下操作 $n-1$ 次：
    • 将堆顶（最大值）与当前堆的最后一个元素交换
    • 堆的大小减1（最大值已就位）
    • 对新的堆顶执行"下沉"调整，恢复堆性质
性质堆排序复杂度分析
  • 建堆时间：$O(n)$（不是 $O(n\log n)$！自底向上建堆是线性的）
  • 排序时间：$O(n \log n)$（每次调整 $O(\log n)$，共 $n-1$ 次）
  • 总时间：$O(n \log n)$（最好 = 平均 = 最坏）
  • 空间复杂度：$O(1)$（原地排序）
  • 稳定性：不稳定
例题建最大堆手工模拟

对序列 {45, 80, 55, 40, 42, 85}（共6个元素），建立最大堆：

数组下标从0开始，最后一个非叶节点 = $\lfloor(6-2)/2\rfloor = 2$

  1. 调整下标2（值55）：
    子节点：index5（值85）。85 > 55，交换
    数组变为：{45, 80, 85, 40, 42, 55}
  2. 调整下标1（值80）：
    子节点：index3（值40），index4（值42）。80 > 40且80 > 42，不交换
    数组不变：{45, 80, 85, 40, 42, 55}
  3. 调整下标0（值45）：
    子节点：index1（值80），index2（值85）。max(80,85)=85 > 45，与85交换
    数组变为：{85, 80, 45, 40, 42, 55}
    继续下沉index2（值45）：子节点index5（值55）。55 > 45，交换
    数组变为：{85, 80, 55, 40, 42, 45}

最终最大堆：{85, 80, 55, 40, 42, 45}

对应的树形结构：

        85
       /  \
     80    55
    / \    /
   40  42 45
洞察堆排序 vs 快速排序

找前k大/小的问题：堆排序只需建堆 $O(n)$ + 取 $k$ 次堆顶 $O(k\log n)$，总计 $O(n + k\log n)$。当 $k \ll n$ 时远优于快排的 $O(n\log n)$。

例：从1000个元素中找前5小，用最小堆只需约 $1000 + 5\times10 = 1050$ 次操作。

6.9多关键字排序

定义多关键字排序问题

当记录有多个关键字（如 key1, key2）时，要求：先按主关键字key1排序；key1相同的，再按次关键字key2排序。

两种策略：

  • MSD（Most Significant Digit first，最高位优先）：先按主关键字分组，组内再按次关键字排序
  • LSD（Least Significant Digit first，最低位优先）：先按次关键字排序，再按主关键字排序
性质LSD方法对稳定性的要求

关键结论：LSD方法要求最后一趟排序必须是稳定的。

原因：LSD先按次关键字排好序，然后按主关键字排序。如果第二趟排序不稳定，可能打乱第一趟已经排好的次关键字顺序。只有稳定排序才能保证"主关键字相同的元素，保持次关键字的有序性"。

例题多关键字排序策略选择

问题：线性表中每个元素有key1和key2两个数据项，要求按key1非递减排序，key1相同时按key2非递减排序。以下哪种方法满足要求？

A: 先按key1冒泡，再按key2选择排序
B: 先按key2冒泡，再按key1选择排序
C: 先按key1选择排序，再按key2冒泡
D: 先按key2选择排序，再按key1冒泡排序

分析（LSD思路）：先排次关键字key2，再排主关键字key1。最后一趟必须用稳定排序。

  • D：先按key2选择排序（不稳定，但这是第一趟，对key2排序），再按key1冒泡排序（稳定！最后一趟稳定）
  • 冒泡排序稳定 → key1相同的元素不会改变相对顺序 → 保持了key2的有序性

答案：D

易错LSD中第一趟排序不要求稳定

LSD方法中，第一趟（对次关键字排序）不要求稳定——因为其结果只是"在主关键字相同的记录中保持有序"，而这一性质由第二趟的稳定性来保证。

但如果有3个以上关键字，则除了最后一趟必须稳定外，中间各趟也必须稳定。

  • 堆排序与归并排序：最坏都是 $O(n\log n)$
  • 冒泡排序与插入排序：最坏都是 $O(n^2)$

注意：快速排序与堆排序的最坏复杂度不同（$O(n^2)$ vs $O(n\log n)$）。

洞察如何记忆稳定性

稳定的排序：冒泡、插入、归并 —— 口诀"冒插归"（帽子龟）

不稳定的排序：选择、希尔、快速、堆 —— 口诀"选希快堆"（选稀快堆）

选排序算法的一般原则：(1) 数据量小或基本有序 → 插入排序；(2) 数据量大且内存充足 → 快排（平均最快）；(3) 要求最坏也保证 $O(n\log n)$ → 堆排序或归并排序；(4) 外部排序 → 归并排序；(5) 找前k大/小 → 堆排序。

对序列 {H, E, B, L, G, A, F, J, I, C, D, K} 中的关键码按字母顺序的升序重新排列，则：
a) 冒泡排序第一趟交换结束后的结果是？（1分）
b) 二路归并排序第一趟归并后的结果是？（1分）
c) 初始步长为4的希尔（shell）排序第一趟后的结果是？（2分）

原始序列（12个元素）：H, E, B, L, G, A, F, J, I, C, D, K

字母顺序：A<B<C<D<E<F<G<H<I<J<K<L

a) 冒泡排序第一趟（从左往右冒，大的往右）

  • (H,E): H>E, 交换 → E,H,B,L,G,A,F,J,I,C,D,K
  • (H,B): H>B, 交换 → E,B,H,L,G,A,F,J,I,C,D,K
  • (H,L): H<L, 不换
  • (L,G): L>G, 交换 → E,B,H,G,L,A,F,J,I,C,D,K
  • (L,A): L>A, 交换 → E,B,H,G,A,L,F,J,I,C,D,K
  • (L,F): L>F, 交换 → E,B,H,G,A,F,L,J,I,C,D,K
  • (L,J): L>J, 交换 → E,B,H,G,A,F,J,L,I,C,D,K
  • (L,I): L>I, 交换 → E,B,H,G,A,F,J,I,L,C,D,K
  • (L,C): L>C, 交换 → E,B,H,G,A,F,J,I,C,L,D,K
  • (L,D): L>D, 交换 → E,B,H,G,A,F,J,I,C,D,L,K
  • (L,K): L>K, 交换 → E,B,H,G,A,F,J,I,C,D,K,L
a) 冒泡第一趟结果：{E, B, H, G, A, F, J, I, C, D, K, L}

b) 二路归并排序第一趟

将相邻元素两两一组进行归并（升序）：

  • (H, E) → {E, H}
  • (B, L) → {B, L}
  • (G, A) → {A, G}
  • (F, J) → {F, J}
  • (I, C) → {C, I}
  • (D, K) → {D, K}
b) 归并第一趟结果：{E, H, B, L, A, G, F, J, C, I, D, K}

洞察备考要点总结
  1. 必背：7种排序的时间/空间复杂度和稳定性（见总结表）
  2. 必练：快速排序划分过程、建堆过程、希尔排序分组过程、归并排序逐趟结果
  3. 必理解：
    • 多关键字排序中LSD对稳定性的要求
    • 正序时插入排序 vs 快排的对比
    • 外部排序为什么选归并排序
    • 找前k大/小为什么选堆排序
  4. 易错点：
    • 选择排序的比较次数与输入无关（始终 $n(n-1)/2$）
    • 快排最坏 $O(n^2)$ 但平均最快
    • 建堆是 $O(n)$ 不是 $O(n\log n)$
    • 希尔排序不稳定，且正序不一定比逆序快
