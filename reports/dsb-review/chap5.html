第五章 查找

高效查找是数据结构的核心目标之一。本章从最朴素的顺序查找出发，依次介绍二分查找、散列表、串匹配等经典算法，着重分析各算法的适用条件、时间复杂度和平均查找长度。

5.0查找概述

定义查找 (Searching)

在数据集合中，根据给定的某个值（关键码 / Key），确定一个其关键码等于给定值的记录或数据元素。

  • 查找表：由同一类型的数据元素（或记录）构成的集合。
  • 关键码：数据元素中能唯一标识该元素的数据项。
  • 查找成功：找到关键码等于给定值的记录。
  • 查找失败：表中不存在关键码等于给定值的记录。
定义平均查找长度 (ASL, Average Search Length)

为确定记录在查找表中的位置，需和给定值进行比较的关键码个数的期望值：

$$\text{ASL} = \sum_{i=1}^{n} p_i \cdot c_i$$

其中 $p_i$ 是查找第 $i$ 个记录的概率，$c_i$ 是找到第 $i$ 个记录所需的比较次数。ASL 是衡量查找算法效率的最重要指标。

5.1顺序查找（Sequential Search）

定义顺序查找

从线性表的一端开始，逐个将元素的关键码与给定值进行比较。若某个元素的关键码等于给定值，则查找成功；若扫描完整个表仍未找到，则查找失败。

算法思想

  1. 从表的第一个元素（或最后一个元素）开始。
  2. 将当前元素的关键码与目标值 key 比较。
  3. 若相等，返回该元素位置（查找成功）。
  4. 若不相等，移动到下一个元素，重复步骤 2-3。
  5. 若扫描完整个表仍未找到，返回查找失败。

平均查找长度分析

定理等概率顺序查找的 ASL

设表长为 $n$，每个元素被查找的概率相等（$p_i = 1/n$），则查找成功的平均查找长度为：

$$\text{ASL}_{\text{succ}} = \sum_{i=1}^{n} \frac{1}{n} \cdot i = \frac{1}{n} \cdot \frac{n(n+1)}{2} = \frac{n+1}{2}$$

即平均需要比较约一半的元素。

直觉不等概率时的优化

如果各元素被查找的概率不等，可以将查找概率大的元素放在前面（靠近表头），这样高频访问的元素更早被找到，从而减少平均比较次数。

例：若 $p_1 \geq p_2 \geq \cdots \geq p_n$，则：

$$\text{ASL} = 1 \cdot p_1 + 2 \cdot p_2 + \cdots + n \cdot p_n$$

这是所有排列中 ASL 最小的情况。

注记时间复杂度
  • 最好：$O(1)$（第一个就是目标）
  • 最坏：$O(n)$（最后一个才是目标，或查找失败）
  • 平均：$O(n)$

适用场景：顺序查找对存储结构没有要求（数组、链表皆可），但效率较低，适用于小规模数据或无序数据。

5.2二分查找（折半查找，Binary Search）

定义二分查找

在一个有序的顺序表（数组）中查找给定值：每次取中间元素与目标比较，若相等则成功；若目标小于中间元素，则在前半部分继续查找；若目标大于中间元素，则在后半部分继续查找。不断将搜索范围缩小一半。

易错点二分查找的前提条件

二分查找有两个必要前提：

  1. 顺序存储（数组）：因为需要通过下标直接访问中间元素（随机访问）。
  2. 元素有序：按关键码值排好序。

链表不能进行二分查找，因为链表无法在 $O(1)$ 时间内访问中间元素。这是历年考试的高频考点！

算法实现

算法二分查找（非递归版）
def binary_search(arr, key):
    low, high = 0, len(arr) - 1
    while low <= high:
        mid = (low + high) // 2
        if arr[mid] == key:
            return mid          # 查找成功
        elif arr[mid] > key:
            high = mid - 1      # 在前半段继续
        else:
            low = mid + 1       # 在后半段继续
    return -1                   # 查找失败
算法二分查找（递归版）
def binary_search_recursive(arr, key, low, high):
    if low > high:
        return -1               # 查找失败
    mid = (low + high) // 2
    if arr[mid] == key:
        return mid
    elif arr[mid] > key:
        return binary_search_recursive(arr, key, low, mid - 1)
    else:
        return binary_search_recursive(arr, key, mid + 1, high)

递归版在每次调用时将搜索范围严格缩半，递归深度即为栈深度。

判定树分析

直觉二分查找的判定树

可以用一棵二叉树（判定树 / Decision Tree）来描述二分查找过程：

  • 根结点 = 第一次比较的中间元素
  • 左子树 = 对前半段继续二分的过程
  • 右子树 = 对后半段继续二分的过程

查找某个元素的比较次数 = 该元素在判定树中的层数。

对于 $n$ 个元素，判定树是一棵深度为 $\lfloor \log_2 n \rfloor + 1$ 的近似完全二叉树。

时间复杂度

定理二分查找的最坏比较次数

对长度为 $n$ 的有序表进行二分查找，最坏情况下的比较次数为：

$$\lfloor \log_2 n \rfloor + 1 = \lceil \log_2(n+1) \rceil$$

直觉：每次比较将搜索空间缩小一半，最多将 $n$ 个元素缩小到 1 个需要 $\lfloor \log_2 n \rfloor$ 次除以 2，加上最后一次比较。

定理递归二分查找的栈深度

使用递归实现二分查找时，递归调用栈的最小容量（最大递归深度）为：

$$\lceil \log_2(N+1) \rceil$$

原因：最坏情况下递归深度等于判定树的高度。每次递归将问题规模从 $N$ 缩小到约 $N/2$，直到子问题规模为 0 或 1。递归树的高度恰好是 $\lceil \log_2(N+1) \rceil$。

二分查找时间复杂度为 $O(\log n)$，空间复杂度为 $O(1)$（非递归）或 $O(\log n)$（递归），要求顺序存储+有序。

5.3二叉搜索树（BST）简述

定义二叉搜索树 (Binary Search Tree, BST)

二叉搜索树是一棵二叉树，满足以下性质：

  • 若左子树非空，则左子树上所有结点的关键码均小于根结点的关键码。
  • 若右子树非空，则右子树上所有结点的关键码均大于根结点的关键码。
  • 左右子树各自也是二叉搜索树。
直觉BST 查找路径的判别

在 BST 中查找一个值 $x$ 时，从根出发，沿路径上的结点值必须满足：

  • 若当前结点值 $v > x$，则下一步走左子树，且后续所有值必须 $< v$。
  • 若当前结点值 $v < x$，则下一步走右子树，且后续所有值必须 $> v$。

这意味着：在查找路径中，已经"往左走过的祖先"提供了上界，"往右走过的祖先"提供了下界。路径上的每个值都必须在这些上下界之间。如果一个序列违反了这个约束，则它不可能是 BST 中的查找路径。

易错点判断查找序列的合法性

给定一个声称是"BST中查找某值的路径序列"，验证其合法性的方法：

  1. 维护当前允许的值范围 $(\text{lower}, \text{upper})$，初始为 $(-\infty, +\infty)$。
  2. 对路径中每个值 $v$：检查 $v$ 是否在 $(\text{lower}, \text{upper})$ 内。
  3. 若 $v > \text{target}$（往左走），更新 $\text{upper} = v$。
  4. 若 $v < \text{target}$（往右走），更新 $\text{lower} = v$。
  5. 若某步 $v$ 超出范围，该序列不合法。

5.4散列表（哈希表，Hash Table）

定义散列表

散列表是根据关键码值直接进行访问的数据结构。通过一个散列函数 $H(\text{key})$，将关键码映射到表中的一个位置（地址），从而实现快速存取。

核心思想：用计算代替比较——理想情况下，一次计算就能确定元素位置，时间复杂度 $O(1)$。

定义冲突（碰撞，Collision）

当两个不同的关键码 $k_1 \neq k_2$，经散列函数计算得到相同的地址，即 $H(k_1) = H(k_2)$，称发生了冲突。$k_1$ 和 $k_2$ 互称为同义词。

定义装填因子（Load Factor）

$$\alpha = \frac{\text{表中元素数}}{\text{表长}(m)}$$

装填因子反映散列表的"拥挤程度"。$\alpha$ 越大，冲突越多，查找效率越低。散列表的 ASL 主要取决于 $\alpha$，而非元素个数 $n$。

散列函数

定义除留余数法（Division Method）

最常用的散列函数构造方法：

$$H(\text{key}) = \text{key} \bmod m$$

其中 $m$ 为散列表的长度。

选取原则：$m$ 最好取素数（质数），这样能使关键码的分布更均匀，减少冲突。

直觉为什么 m 取素数好？

如果 $m$ 有较多的因子，则具有这些因子的关键码更容易映射到相同的地址。取素数时，关键码对 $m$ 取余的结果分布最均匀，"不偏向"任何特定的地址。

冲突处理方法

方法一：开放定址法（Open Addressing）

定义开放定址法

当发生冲突时，按照某种探查序列，在散列表中寻找下一个空闲位置。通用公式：

$$H_i = (H(\text{key}) + d_i) \bmod m, \quad i = 1, 2, 3, \ldots$$

其中 $d_i$ 是第 $i$ 次探查的增量，不同的 $d_i$ 取法构成不同的探查方法。

方法线性探查法（Linear Probing）

$d_i = 1, 2, 3, \ldots$，即依次探查 $H(\text{key})+1, H(\text{key})+2, \ldots$（取模 $m$）。

优点：实现简单。

缺点：容易产生堆积（Clustering）现象——已有的连续占用区域会越来越大，导致后续元素越来越难找到空位，查找性能恶化。

方法二次探查法（Quadratic Probing）

$d_i = 1^2, -1^2, 2^2, -2^2, 3^2, -3^2, \ldots$

可以缓解线性探查的堆积问题，但不能保证表未满时一定能找到空位。

方法再散列法 / 双散列法（Double Hashing）

使用第二个散列函数 $H_2(\text{key})$ 计算探查步长：

$$d_i = i \cdot H_2(\text{key}), \quad i = 1, 2, 3, \ldots$$

探查序列为：$H(\text{key}), H(\text{key}) + H_2(\text{key}), H(\text{key}) + 2 \cdot H_2(\text{key}), \ldots$（均取模 $m$）。

优点：不同关键码的探查序列不同，有效避免堆积。

常见选取：$H_2(\text{key}) = \text{key} \bmod q$（$q < m$，$q$ 为素数）。

方法二：拉链法（链地址法，Chaining）

定义拉链法

将所有散列地址相同的记录（同义词）存储在同一个链表中。散列表的每个位置存放一个链表头指针，冲突的元素依次插入该链表。

优点：

  • 无堆积问题，不同散列地址的元素互不影响。
  • 表长不必事先确定，动态分配。
  • 删除操作简便（只需在链表中摘除结点）。

散列表的查找过程

  1. 计算 $H(\text{key})$，得到散列地址。
  2. 检查该地址：
    • 若为空，则查找失败。
    • 若该位置的关键码等于 key，则查找成功。
    • 若不等（发生冲突），则按冲突处理方法计算下一地址，继续比较。
  3. 重复步骤 2，直到找到目标或确认查找失败。

平均查找长度与装填因子

定理散列表的 ASL 与 α 的关系

散列表的平均查找长度不是表中元素数 $n$ 的函数，而主要取决于装填因子 $\alpha$。

对于不同的冲突处理方法，ASL 的近似公式为：

冲突处理方法查找成功的 ASL查找失败的 ASL
线性探查$\frac{1}{2}\left(1+\frac{1}{1-\alpha}\right)$$\frac{1}{2}\left(1+\frac{1}{(1-\alpha)^2}\right)$
二次探查/双散列$-\frac{1}{\alpha}\ln(1-\alpha)$$\frac{1}{1-\alpha}$
拉链法$1 + \frac{\alpha}{2}$$\alpha + e^{-\alpha}$

可见：$\alpha$ 越小（表越空），ASL 越低，查找越快。通常建议 $\alpha$ 控制在 0.7 ~ 0.8 以内。

易错点影响 ASL 的因素

散列表的关键码比较次数不仅仅取决于散列函数和冲突处理方法，还取决于：

  • 装填因子 $\alpha$（最重要）
  • 关键码的分布特征
  • 元素的插入顺序（不同插入顺序可能导致不同的冲突链）

考试中经常出判断题："关键码的比较次数仅取决于散列函数与冲突处理方法"——这是错的！

5.5串的模式匹配（Pattern Matching）

定义模式匹配问题

给定目标串（主串）$T$，长度为 $n$；模式串 $P$，长度为 $m$（$m \leq n$）。问：$P$ 是否是 $T$ 的子串？若是，首次出现在什么位置？

朴素模式匹配算法（Brute Force）

算法朴素匹配

思想：从目标串的第 1 个字符开始，依次与模式串的字符进行比较：

  1. 令 $i = 0$（目标串起始位置），$j = 0$（模式串当前位置）。
  2. 若 $T[i] = P[j]$，则 $i$++，$j$++，继续比较下一对字符。
  3. 若 $T[i] \neq P[j]$（失配），则模式串回退到开头（$j = 0$），目标串起始位置后移一位（$i = i - j + 1$），重新开始匹配。
  4. 重复直到 $j = m$（匹配成功）或 $i > n - m$（匹配失败）。
定理朴素匹配的时间复杂度

最坏情况：每次在模式串最后一个字符处失配，需要回溯。

例如：$T = \text{"aaa...aab"}$，$P = \text{"aab"}$。

最多比较次数为 $(n - m + 1) \times m$，时间复杂度为：

$$O(n \times m)$$

最好情况：每次第一个字符就失配，或很快匹配成功，时间 $O(n)$。

注记KMP 算法（了解）

KMP 算法利用已匹配的信息避免回溯，时间复杂度优化到 $O(n + m)$。其核心是预处理模式串得到 next 数组（部分匹配表）。本课程以朴素匹配为主要考点。

真题选择题精选

选择二分查找的前提条件
2024 选择题 二分查找 ★☆☆
对线性表进行二分法查找，其前提条件是（）。
A. 线性表以链接方式存储，并且按关键码值排好序
B. 线性表以链接方式存储，并且按关键码值的检索频率排好序
C. 线性表以顺序方式存储，并且按关键码值排好序
D. 线性表以顺序方式存储，并且按关键码值的检索频率排好序

答案：C

  1. 二分查找需要"随机访问中间元素"，即 $O(1)$ 访问第 $\lfloor(low+high)/2\rfloor$ 个元素 → 必须是顺序存储（数组），排除 A、B。
  2. 二分查找每次用中间元素与目标比较，决定搜索左半还是右半 → 必须按关键码值有序，排除 D（按检索频率排序对二分无意义）。
  3. 故选 C。
C. 线性表以顺序方式存储，并且按关键码值排好序
选择二分查找最坏比较次数
2024 选择题 二分查找复杂度 ★☆☆
在二分查找算法中，每次比较后都将搜索范围缩小一半，若要在长度为 $n$ 的数组中查找一个元素，最坏情况下需要比较多少次（）。
A. $n$   B. $\lfloor n/2 \rfloor$   C. $\lfloor \log n \rfloor$   D. $\lfloor \log n \rfloor + 1$

答案：D

  1. 设搜索范围从 $n$ 开始，每次比较后缩小为约 $n/2$。
  2. 经过 $k$ 次比较后，剩余元素数约为 $n/2^k$。当 $n/2^k < 1$ 时停止。
  3. 解 $2^k > n$ 得 $k > \log_2 n$，即 $k_{\min} = \lfloor \log_2 n \rfloor + 1$。
  4. 等价表达式为 $\lceil \log_2(n+1) \rceil$。
D. $\lfloor \log_2 n \rfloor + 1$
选择递归二分查找的栈容量
2022 选择题 递归/栈深度 ★★☆
给定一个 $N$ 个相异元素构成的有序数列，设计一个递归算法实现数列的二分查找，考察递归过程中栈的使用情况，请问这样一个递归调用栈的最小容量应为（）。
A. $N$   B. $N/2$   C. $\lfloor \log_2 N \rfloor$   D. $\lceil \log_2(N+1) \rceil$

答案：D

  1. 递归二分查找每次将问题规模从 $N$ 缩小到约 $N/2$（严格地说是 $\lfloor(N-1)/2\rfloor$ 或 $\lceil(N-1)/2\rceil$）。
  2. 递归终止条件是子数组为空（low > high）。
  3. 最深的递归路径即为最坏情况的查找路径，长度等于判定树高度。
  4. $N$ 个结点的判定树高度 = $\lceil \log_2(N+1) \rceil$。这就是所需的最小栈容量。
  5. 为什么不是 $\lfloor \log_2 N \rfloor$？举例：$N = 4$ 时，$\lfloor \log_2 4 \rfloor = 2$，但实际递归深度可达 3 层（mid→左半mid→再半），即 $\lceil \log_2 5 \rceil = 3$。
D. $\lceil \log_2(N+1) \rceil$

  • 363 在 $(344, 397)$ 内 → 找到！

序列 B：925, 202, 911, 240, 912, 245, 363

填空朴素模式匹配的最多比较次数
2022 填空题 串匹配 ★☆☆
目标串长是 $n$，模式串长是 $m$，朴素模式匹配算法思想为：从目标串第一个字符开始，依次与模式串字符匹配；若匹配失败，则尝试匹配的目标串起始字符位置往后移一位，重新开始依次和模式串字符匹配；......；直到匹配成功或遍历完整个目标串为止。则该算法中字符的最多比较次数是____（使用大O表示法）。

答案：$O(n \times m)$

  1. 目标串有 $n - m + 1$ 个可能的起始位置。
  2. 对于每个起始位置，最坏情况下需要与模式串比较 $m$ 个字符才能判定失配（比如前 $m-1$ 个字符都匹配，第 $m$ 个失配）。
  3. 总比较次数 $\leq (n - m + 1) \times m$，用大 O 表示为 $O(nm)$。
$O(n \times m)$
填空不等概率顺序查找的平均比较次数
2022 填空题 顺序查找/ASL ★★☆
对长度为 3 的顺序表进行查找，若查找第一个元素的概率为 $1/2$，查找第二个元素的概率为 $1/4$，查找第三个元素的概率为 $1/8$，则执行任意查找需要比较元素的平均个数为____。

答案：$11/8$

  1. 各元素的查找概率：$p_1 = 1/2$，$p_2 = 1/4$，$p_3 = 1/8$。
  2. 各元素的比较次数：查找第 1 个元素需比较 1 次，第 2 个需 2 次，第 3 个需 3 次。
  3. 平均比较次数： $$\text{ASL} = 1 \times \frac{1}{2} + 2 \times \frac{1}{4} + 3 \times \frac{1}{8} = \frac{1}{2} + \frac{2}{4} + \frac{3}{8} = \frac{4}{8} + \frac{4}{8} + \frac{3}{8} = \frac{11}{8}$$

注：题目中三个概率之和 $= 1/2 + 1/4 + 1/8 = 7/8 \neq 1$。剩余 $1/8$ 的概率对应查找失败的情况。如果题目要求包含查找失败（失败时也比较 3 次），则 ASL $= 11/8 + 3 \times 1/8 = 14/8 = 7/4$。但本题问的是"执行任意查找需要比较元素的平均个数"，按字面计算直接加权即可。

$\dfrac{11}{8}$
