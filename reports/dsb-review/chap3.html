第三章 树与二叉树

树是非线性结构的核心，二叉树是整门课的重中之重。本章涵盖树的基本概念、二叉树性质与遍历、BST、哈夫曼编码、堆、森林等核心内容。

3.1树的基本概念

定义树（Tree）

递归定义：树是 $n$（$n \ge 0$）个结点的有限集合 $T$。当 $n=0$ 时为空树；当 $n > 0$ 时，存在唯一的根结点 $r$，其余结点可分为 $m$（$m \ge 0$）个互不相交的有限集 $T_1, T_2, \ldots, T_m$，每个集合本身又是一棵树，称为根的子树。

直觉：树是一种"一对多"的层次关系，从根出发向下分裂，每个结点可以有零个或多个后继，但最多只有一个前驱（父结点）。

定义树的术语
  • 根（Root）：没有父结点的结点，整棵树唯一。
  • 叶子（Leaf）/ 终端结点：没有子结点（度为 0）的结点。
  • 度（Degree）：一个结点的子结点个数。树的度 = 所有结点度的最大值。
  • 层（Level）：根结点在第 0 层（或第 1 层，视约定），其子女在第 1 层（或第 2 层），依次类推。
  • 高度（Height）：从根到最远叶子的路径上的边数（或结点数减1）。本课程约定：高度 = 最大层数 + 1（若根在第 0 层）。
  • 深度（Depth）：从根到某结点的路径上的边数（根的深度为 0）。
  • 兄弟：拥有同一父结点的结点互称兄弟。
  • 祖先 / 子孙：从根到某结点路径上的所有结点是该结点的祖先；反之为子孙。
性质树的基本性质

性质 1：$n$ 个结点的树恰好有 $n-1$ 条边。

证明：除根结点外，每个结点恰好有一条边连向其父结点，所以边数 = $n - 1$。

性质度与结点数的关系（高频考点）

设一棵树中，度为 $i$ 的结点有 $n_i$ 个（$i = 0, 1, 2, \ldots, m$），则：

$$n = n_0 + n_1 + n_2 + \cdots + n_m$$

边数 = $n - 1$ = 所有结点的度之和 = $\sum_{i=1}^{m} i \cdot n_i$

因此：

$$\boxed{n = 1 + \sum_{i=1}^{m} i \cdot n_i}$$

推论（叶子数公式）：将 $n = n_0 + n_1 + \cdots + n_m$ 代入 $n - 1 = \sum i \cdot n_i$：

$$n_0 + n_1 + \cdots + n_m - 1 = n_1 + 2n_2 + 3n_3 + \cdots + mn_m$$ $$\boxed{n_0 = 1 + \sum_{i=2}^{m}(i-1) \cdot n_i = 1 + n_2 + 2n_3 + 3n_4 + \cdots + (m-1)n_m}$$

直觉：每多一个度为 $k$ 的结点（$k \ge 2$），就比"只有度 0 和度 1"的情况多产生 $k-1$ 个叶子。

例题度为 3 的树中结点计数

在一棵度为 3 的树中，度为 3 的结点有 2 个，度为 2 的结点有 1 个，度为 1 的结点有 0 个，求叶子结点数。

方法一：用公式

$n_0 = 1 + (2-1) \times n_2 + (3-1) \times n_3 = 1 + 1 \times 1 + 2 \times 2 = 1 + 1 + 4 = 6$

方法二：从边数出发

边数 = $3 \times 2 + 2 \times 1 + 1 \times 0 = 8$，所以 $n = 9$。

$n_0 = 9 - 0 - 1 - 2 = 6$。

$n_0 = 6$

3.2二叉树

定义二叉树（Binary Tree）

二叉树是 $n$（$n \ge 0$）个结点的有限集合，或者是空集（$n=0$），或者由一个根结点和两棵互不相交的、分别称为左子树和右子树的二叉树组成。

关键区分：二叉树不是度为 2 的树！二叉树严格区分左右子树，即使只有一个子结点也要区分是左还是右。

二叉树的重要性质

性质 1各层最多结点数

二叉树第 $i$ 层（根为第 0 层）上最多有 $2^i$ 个结点。

证明（归纳法）：第 0 层有 1 = $2^0$ 个结点（根）。设第 $i$ 层最多 $2^i$ 个，则第 $i+1$ 层最多 $2 \times 2^i = 2^{i+1}$ 个。

性质 2最多结点总数

高度为 $h$ 的二叉树（根在第 0 层，最大层数为 $h$）最多有 $\displaystyle\sum_{i=0}^{h} 2^i = 2^{h+1} - 1$ 个结点。

性质 3$n_0 = n_2 + 1$（极高频考点）

对任何一棵非空二叉树，若叶子结点数为 $n_0$，度为 2 的结点数为 $n_2$，则：

$$\boxed{n_0 = n_2 + 1}$$

证明：设度为 1 的结点数为 $n_1$。

  • 结点总数 $n = n_0 + n_1 + n_2$
  • 边数 = $n - 1 = n_1 + 2n_2$（每个度为 $k$ 的结点贡献 $k$ 条向下的边）
  • 两式相减：$n_0 + n_1 + n_2 - 1 = n_1 + 2n_2$
  • 化简得 $n_0 = n_2 + 1$。

直觉：树从根开始有 1 个结点，每个度为 2 的结点"分裂"出 1 个额外结点，度为 1 的结点不增不减，所以叶子总比"分裂点"多 1。

性质 4完全二叉树的高度

含 $n$ 个结点的完全二叉树的高度（层数 - 1）为 $h = \lfloor \log_2 n \rfloor$。

若定义高度 = 层数，则高度 = $\lfloor \log_2 n \rfloor + 1$。

完全二叉树与满二叉树

定义满二叉树 / 完全二叉树

满二叉树：所有非叶结点的度都为 2，且所有叶子在同一层。等价地说，没有度为 1 的结点，且所有叶子在最底层。高度 $h$ 的满二叉树恰好有 $2^{h+1} - 1$ 个结点。

完全二叉树：对一棵满二叉树，按层序编号后，从最后一个结点开始连续去掉若干个结点得到的就是完全二叉树。等价定义：除最后一层外每层都是满的，且最后一层的结点集中在左侧。

性质完全二叉树的编号关系

按层序从 0 开始编号（根为 0）：

  • 结点 $i$ 的左孩子编号：$2i + 1$
  • 结点 $i$ 的右孩子编号：$2i + 2$
  • 结点 $i$ 的父结点编号：$\lfloor(i-1)/2\rfloor$

按层序从 1 开始编号（根为 1）：

  • 结点 $i$ 的左孩子编号：$2i$
  • 结点 $i$ 的右孩子编号：$2i + 1$
  • 结点 $i$ 的父结点编号：$\lfloor i/2 \rfloor$
洞察完全二叉树中 $n_1$ 的取值

完全二叉树中度为 1 的结点要么有 0 个（$n$ 为奇数），要么有 1 个（$n$ 为偶数）。

原因：完全二叉树最多只有最后一个非叶子结点可能只有左孩子而无右孩子（度为 1），而这只在总结点数为偶数时发生。

所以对于 $n$ 个结点的完全二叉树：

  • $n$ 为奇数：$n_1 = 0$，$n_0 = (n+1)/2$，$n_2 = (n-1)/2$
  • $n$ 为偶数：$n_1 = 1$，$n_0 = n/2$，$n_2 = n/2 - 1$

统一公式：$n_0 = \lceil n/2 \rceil$。

易错满二叉树的不同定义

国内教材常用的定义：满二叉树 = 每层都满 = 高度 $h$ 恰有 $2^{h+1}-1$ 个结点。

但本课程部分考题中定义"没有度为 1 的结点"的二叉树为满二叉树（更接近英文 full binary tree）。此时满二叉树只要求 $n_1 = 0$，叶子不必全在同一层。若 $n_1=0$，则 $n = n_0 + n_2 = 2n_0 - 1$（总是奇数），叶子数 = $(n+1)/2$。

例题120 个结点的完全二叉树

已知某棵完全二叉树有 120 个结点，求层数和叶子数。

层数：$\lfloor \log_2 120 \rfloor + 1 = 6 + 1 = 7$ 层（根在第 1 层）。

叶子数：$n = 120$ 为偶数，所以 $n_1 = 1$。

由 $n_0 = n_2 + 1$ 和 $n_0 + n_1 + n_2 = 120$：

$n_0 + 1 + (n_0 - 1) = 120 \Rightarrow 2n_0 = 120 \Rightarrow n_0 = 60$

7 层，60 个叶子结点。

3.3二叉树的遍历（核心考点）

定义四种遍历方式
  • 前序遍历（先根）：根 → 左子树 → 右子树
  • 中序遍历（中根）：左子树 → 根 → 右子树
  • 后序遍历（后根）：左子树 → 右子树 → 根
  • 层次遍历：按层从上到下、从左到右访问，用队列实现

根据遍历序列还原二叉树

定理遍历序列确定二叉树
  • 前序 + 中序 → 唯一确定一棵二叉树。
  • 后序 + 中序 → 唯一确定一棵二叉树。
  • 前序 + 后序 → 不能唯一确定（除非每个结点度为 0 或 2）。

还原方法：

  • 前序序列的第一个元素（或后序序列的最后一个元素）一定是根。
  • 在中序序列中找到根，根左边的是左子树的中序，根右边的是右子树的中序。
  • 根据左/右子树中序的长度，在前序/后序中分割出对应的子序列。
  • 对左右子树递归执行上述步骤。
例题由中序 + 后序还原二叉树

中序：DGBAECF，后序：GDBEFCA。求前序遍历序列。

  • 后序最后一个字符 A 是根。在中序中 A 的位置将序列分为：左子树中序 = DGB，右子树中序 = ECF。
  • 后序中对应：左子树后序 = GDB（前3个），右子树后序 = EFC（接下来3个）。
  • 左子树：后序最后字符 B 为根。中序 DGB 中 B 在最右，左子树中序 = DG，右子树为空。
  • 左子树的左子树：后序 GD 最后字符 D 为根。中序 DG 中 D 在最左，左子树为空，右子树中序 = G。
  • 右子树：后序 EFC 最后字符 C 为根。中序 ECF 中 C 分割为左子树中序 = E，右子树中序 = F。

最终树结构：A(左:B(左:D(右:G)), 右:C(左:E, 右:F))

前序遍历：A → B → D → G → C → E → F

前序遍历序列为 ABDGCEF

重要结论

结论 1前序与后序相反的条件

若一棵二叉树的前序遍历序列和后序遍历序列正好相反，则该二叉树中每个结点至多一个孩子，即整棵树是一条"链"（类似线性表）。等价地说，该二叉树的高度等于结点数。

直觉：前序是"根左右"，后序是"左右根"。反过来后后序变成"根右左"。要让"根左右"="根右左"，必须每个结点要么没有子树，要么只有一个子树（此时不存在"左"和"右"的区分问题——但注意这并不要求一定无左或一定无右，只要每个结点至多一个孩子即可）。

注意：本课程 2021 年考题的标准答案是 B（高度等于结点数），因为"任一结点无左孩子"（C）或"任一结点无右孩子"（D）只是充分条件之一，而非充要条件。但"高度=结点数"等价于"每个结点至多一个孩子"，是充要条件。

结论 2叶结点的相对顺序不变

任意一棵二叉树中，所有叶结点在前序、中序、后序遍历中的相对次序不变。

证明：叶结点没有子树，对于任意两个叶结点 $u, v$，它们的相对位置只取决于它们分属哪棵子树的左右关系。前序、中序、后序都是先遍历左子树再遍历右子树，所以左子树中的叶子一定在右子树中的叶子前面，对所有三种遍历都成立。

结论 3前序中X在Y前 且 后序中X在Y后 → X是Y的祖先

设结点 X 和 Y 是二叉树中任意两个结点。若在先根遍历中 X 在 Y 前，在后根遍历中 X 在 Y 后，则 X 是 Y 的祖先。

分析：前序中祖先在子孙前（先访问根再递归）；后序中祖先在子孙后（先递归再访问根）。若 X 不是 Y 的祖先，则 X 与 Y 要么是兄弟子树关系（此时两种遍历中相对顺序相同），要么 Y 是 X 的祖先（此时前序中 Y 在 X 前），均矛盾。

洞察3 个结点的不同二叉树

3 个结点可以构成 5 种不同形态的二叉树（第 $n$ 个 Catalan 数 $C_3 = 5$）。

若前序确定为 ADC，则 A 必为根，D 和 C 的位置有 5 种不同排列方式对应 5 种不同的二叉树。

3.4二叉排序树（BST）

定义二叉排序树 / 二叉搜索树（Binary Search Tree）

二叉排序树或者是空树，或者满足以下性质的二叉树：

  • 左子树上所有结点的关键字 小于 根结点的关键字；
  • 右子树上所有结点的关键字 大于 根结点的关键字；
  • 左子树和右子树各自也是二叉排序树。
性质BST 的中序遍历

BST 的中序遍历序列是严格递增的有序序列。

逆命题：若给定一个序列是某棵二叉树的中序遍历结果，当且仅当该序列严格递增时，该二叉树可能是 BST。

BST 的操作

操作查找

从根开始比较：等于则找到；小于则进入左子树；大于则进入右子树。时间复杂度 $O(h)$，$h$ 为树高。

操作插入

按查找路径走到空位置处，将新结点作为叶子插入。插入后仍满足 BST 性质。

操作删除（重点）

删除结点 $p$ 分三种情况：

  • $p$ 是叶子结点：直接删除，父结点对应指针置空。
  • $p$ 只有一个子树：用该子树替代 $p$ 的位置。
  • $p$ 有两个子树：用 $p$ 的中序前驱（左子树中最大的结点）或中序后继（右子树中最小的结点）替代 $p$，然后删除那个前驱/后继（它最多只有一个子树，递归到前两种情况）。
例题BST 删除结点 37

在以下 BST 中删除结点 37（用左子树最大者替代）：

树结构：37 为根，左子树 24(左 7(左 2)，右 32(左 30))，右子树 42(左 40，右 120)

左子树最大者 = 从 24 出发一直向右，到达 32。

用 32 替代 37 的位置。32 原来有左子 30，30 替代 32 原来的位置（挂到 24 的右边）。

删除后的树：

32 为根，左子树 24(左 7(左 2)，右 30)，右子树 42(左 40，右 120)

32 替代 37 成为新根，30 上移到原来 32 的位置。
易错BST 不一定是满二叉树

BST 的形态取决于插入顺序。如果按升序插入 1,2,3,4,5，得到的 BST 是一条向右的链，退化为线性结构。BST 不一定是满二叉树，也不一定是完全二叉树。

3.5哈夫曼树与哈夫曼编码

定义带权路径长度 WPL

设二叉树有 $n$ 个叶子结点，第 $i$ 个叶子结点的权值为 $w_i$，从根到该叶子的路径长度（边数）为 $l_i$，则该树的带权路径长度为：

$$WPL = \sum_{i=1}^{n} w_i \times l_i$$
定义哈夫曼树（最优二叉树）

在所有含有相同叶子结点（权值确定）的二叉树中，WPL 最小的那棵称为哈夫曼树。

哈夫曼树的构建算法

算法哈夫曼算法（贪心）
  • 将所有权值看作独立的单结点树，放入集合 $S$。
  • 从 $S$ 中取出权值最小的两棵树 $T_1, T_2$，将它们作为左右子树构建新树，新树的根权值 = $w_1 + w_2$。
  • 将新树放回 $S$。
  • 重复步骤 2-3，直到 $S$ 中只剩一棵树，即为哈夫曼树。

性质：哈夫曼树中没有度为 1 的结点（只有度为 0 和度为 2 的结点）。

定义哈夫曼编码

利用哈夫曼树对字符进行编码：从根出发，走左分支记为 0，走右分支记为 1，到达叶子时得到的 0/1 串就是该字符的编码。

前缀码性质：哈夫曼编码是前缀码（prefix code），即没有任何一个字符的编码是另一个字符编码的前缀，因此解码时不会产生歧义。

编码总长度：对电文编码的总长度恰好等于哈夫曼树的 WPL（这里 $w_i$ = 字符出现次数，$l_i$ = 该字符编码的位数）。

例题构建哈夫曼树（权值 4,8,6,9,18）

构建过程：

  • 初始集合：{4, 6, 8, 9, 18}
  • 取最小两个 4 和 6，合并为 10。集合变为 {8, 9, 10, 18}
  • 取最小两个 8 和 9，合并为 17。集合变为 {10, 17, 18}
  • 取最小两个 10 和 17，合并为 27。集合变为 {18, 27}
  • 取 18 和 27，合并为 45。完成。

树结构：

         45
        /  \
      18    27
           /  \
         10    17
        / \   / \
       4   6 8   9

计算 WPL：

18 在第 1 层（路径长度 1），4/6/8/9 在第 3 层（路径长度 3）：

$WPL = 18 \times 1 + 4 \times 3 + 6 \times 3 + 8 \times 3 + 9 \times 3$

$= 18 + 12 + 18 + 24 + 27 = 99$

WPL = 99
例题哈夫曼编码电文长度

字符 a 出现 1000 次，b 出现 2000 次，c 出现 6000 次，d 出现 1000 次。求哈夫曼编码后电文总长度。

权值集合：{a:1000, d:1000, b:2000, c:6000}

  • 取 a(1000) + d(1000) = 2000。集合：{2000, b:2000, c:6000}
  • 取 2000 + b(2000) = 4000。集合：{4000, c:6000}
  • 取 4000 + c(6000) = 10000。

树结构：c 在第 1 层，b 在第 2 层，a 和 d 在第 3 层。

编码长度：c = 1 位，b = 2 位，a = 3 位，d = 3 位。

总长度 = $6000 \times 1 + 2000 \times 2 + 1000 \times 3 + 1000 \times 3 = 6000 + 4000 + 3000 + 3000 = 16000$ 位

电文总长度为 16000 个比特

3.6堆（Heap）

定义堆

最小堆（小根堆）：完全二叉树，每个结点的值 $\le$ 其子结点的值。根结点是全局最小值。

最大堆（大根堆）：完全二叉树，每个结点的值 $\ge$ 其子结点的值。根结点是全局最大值。

存储：利用完全二叉树的性质，用数组存储。编号从 0 开始，结点 $i$ 的左孩子为 $2i+1$，右孩子为 $2i+2$，父结点为 $\lfloor(i-1)/2\rfloor$。

堆的核心操作

操作建堆（Build Heap）—— O(n)

给定一个无序数组，将其调整为堆：

  • 从最后一个非叶结点开始（编号为 $\lfloor n/2 \rfloor - 1$），向前逐个执行下沉（sift down）操作。
  • 下沉操作：将当前结点与其较小（小根堆）/ 较大（大根堆）的子结点比较，若不满足堆序性质则交换，继续向下直到满足条件或到达叶子。

时间复杂度：$O(n)$（不是 $O(n \log n)$！）。直觉：底层结点多但下沉距离短，顶层结点少但下沉距离长，总和是 $O(n)$。

操作插入 —— O(log n)

将新元素添加到数组末尾（完全二叉树的最后一个位置），然后执行上浮（sift up）：将其与父结点比较，若不满足堆序性质则交换，一直向上直到满足条件或到达根。

操作删除最小/最大 —— O(log n)
  • 将根（最小/最大元素）取出。
  • 用数组最后一个元素替换根的位置。
  • 对新的根执行下沉（sift down）操作，恢复堆序。
易错建堆复杂度不是 O(n log n)

很多同学误以为建堆是 $O(n \log n)$。实际上自底向上建堆只需 $O(n)$。这是因为大部分结点在底层，它们下沉的距离很短。精确分析：$\sum_{h=0}^{H} \lceil n/2^{h+1} \rceil \cdot h = O(n)$。

3.7森林

定义森林（Forest）

森林是 $m$（$m \ge 0$）棵互不相交的树的集合。

森林与二叉树的互相转换

方法森林 → 二叉树（左孩子右兄弟表示法）
  • 将每棵树用"左孩子-右兄弟"表示法转化为二叉树：结点的第一个孩子作为其左子结点，结点的下一个兄弟作为其右子结点。
  • 将各棵树的根通过"右兄弟"链接起来：第一棵树的根是二叉树的根，第二棵树的根是第一棵树根的右子结点，第三棵树的根是第二棵树根的右子结点……

关键结论：森林转化为二叉树后，二叉树根的右子树对应除第一棵树外的其余森林。

性质森林的遍历与对应二叉树的遍历
  • 森林的先序遍历 = 对应二叉树的先序遍历
  • 森林的中序遍历（也叫后根遍历） = 对应二叉树的中序遍历

森林的先序遍历：访问第一棵树的根 → 先序遍历第一棵树的子树森林 → 先序遍历其余树组成的森林。

森林的中序遍历：中序遍历第一棵树的子树森林 → 访问第一棵树的根 → 中序遍历其余树组成的森林。

性质森林中度与结点数的关系

设森林 $F$ 中有 $T$ 棵树，叶子结点总数为 $L$，度为 2 的结点总数为 $N_2$。对于每棵树单独看：

每棵树 $n_i$ 个结点有 $n_i - 1$ 条边，森林总边数 = $\sum(n_i - 1) = N - T$（$N$ 为总结点数）。

又总边数 = $\sum_{i=1}^{m} i \cdot N_i$，其中 $N_i$ 是度为 $i$ 的结点数。

若森林中每个结点的度不超过 2（子结点个数 ≤ 2），则：

$N - T = N_1 + 2N_2$，且 $N = L + N_1 + N_2$，解得 $\boxed{T = L - N_2}$。
