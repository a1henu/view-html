第一章 线性表（顺序表与链表）

线性表是最基础也最重要的数据结构之一。掌握顺序表和链表的原理与操作，是理解后续栈、队列、树、图等复杂结构的必备前提。本章从零开始，详细讲解线性表的概念、两种存储方式、各项操作的时间复杂度，以及如何在 Python 中实现链表。

1.1线性表的定义与基本概念

定义线性表 (Linear List)

线性表是由 $n$（$n \geq 0$）个相同类型的数据元素组成的有限序列。记作：

$$L = (a_1, a_2, \ldots, a_n)$$

其中：

  • $n$ 称为线性表的长度；当 $n = 0$ 时称为空表。
  • $a_1$ 称为表头元素（没有前驱），$a_n$ 称为表尾元素（没有后继）。
  • 对于 $1 < i < n$，元素 $a_i$ 有且仅有一个直接前驱 $a_{i-1}$ 和一个直接后继 $a_{i+1}$。
直觉如何理解"线性"？

想象一列火车车厢：每节车厢（元素）只与前一节和后一节相连，不会有分叉（这点和"树"不同），也不会有环路（和"图"不同）。这种"一对一"的前后关系就是线性结构的本质。

与之对比：

  • 树形结构：一对多（一个父结点可以有多个子结点）
  • 图形结构：多对多（任意结点之间都可能有边）

线性表的基本操作（ADT）

作为抽象数据类型（ADT），线性表通常需要支持以下操作：

操作名称功能描述
InitList()初始化一个空的线性表
Length()返回线性表中元素的个数
Get(i)获取第 $i$ 个位置的元素（按位查找）
Locate(x)查找值为 $x$ 的元素，返回其位置（按值查找）
Insert(i, x)在第 $i$ 个位置插入元素 $x$
Delete(i)删除第 $i$ 个位置的元素
IsEmpty()判断线性表是否为空
注记逻辑结构 vs 存储结构

上面的定义和操作描述的是线性表的逻辑结构——即"元素之间的关系是什么"。而如何在计算机内存中真正存放这些元素，则是存储结构（物理结构）的问题。同一种逻辑结构可以有不同的存储方式：

  • 顺序存储 → 顺序表（数组）
  • 链式存储 → 链表

1.2顺序表（数组实现）

存储原理：连续内存

定义顺序表 (Sequential List / Array)

顺序表用一段地址连续的存储单元依次存储线性表中的数据元素。

设第一个元素 $a_0$ 的存储地址为 $\text{LOC}(a_0)$，每个元素占 $d$ 个存储单元，则第 $i$ 个元素的地址为：

$$\text{LOC}(a_i) = \text{LOC}(a_0) + i \times d$$
直觉为什么能随机访问？

想象一排连续编号的储物柜：如果你知道第 0 号柜子的位置（LOC(a_0)），以及每个柜子的宽度（$d$），那么要找第 $i$ 号柜子，你只需要做一次简单的乘法和加法——不需要从头一个个数过去。

这就是随机访问（Random Access）的含义：通过下标直接计算出地址，时间复杂度 $O(1)$。

在 Python 中，内置的 list 就是用连续数组实现的顺序表（底层为动态数组 / dynamic array）。

插入操作

定理顺序表插入的平均时间复杂度

在长度为 $n$ 的顺序表中，在第 $i$ 个位置（$0 \le i \le n$）插入一个新元素，需要将后面的 $n - i$ 个元素依次后移一位。

若在每个位置插入的概率相同（均为 $\frac{1}{n+1}$），则平均移动元素次数为：

$$E = \sum_{i=0}^{n} \frac{1}{n+1} \cdot (n - i) = \frac{1}{n+1} \cdot \frac{n(n+1)}{2} \cdot \frac{1}{1} = \frac{n}{2}$$

因此，插入操作的平均时间复杂度为 $O(n)$。

点击展开：平均移动次数的推导

设在位置 $i$ 插入（$i = 0, 1, \ldots, n$，共 $n+1$ 种可能），需要移动 $n - i$ 个元素。

假设在各位置插入的概率相等，即每个概率为 $p_i = \frac{1}{n+1}$，则：

$$E = \sum_{i=0}^{n} p_i \cdot (n-i) = \frac{1}{n+1}\sum_{i=0}^{n}(n-i)$$

令 $k = n - i$，当 $i$ 从 0 到 $n$ 时，$k$ 从 $n$ 到 0：

$$E = \frac{1}{n+1}\sum_{k=0}^{n} k = \frac{1}{n+1} \cdot \frac{n(n+1)}{2} = \frac{n}{2}$$

所以平均需要移动 $\frac{n}{2}$ 个元素，时间复杂度为 $O(n)$。

例题在顺序表 [10, 20, 30, 40, 50] 的位置 2 插入元素 25

初始状态（下标从 0 开始）：

下标:  0    1    2    3    4
元素: [10] [20] [30] [40] [50]

在位置 2 插入 25，需要把下标 2、3、4 的元素分别后移一位：

  1. 先将 a[4]=50 移到 a[5]
  2. 将 a[3]=40 移到 a[4]
  3. 将 a[2]=30 移到 a[3]
  4. 将 25 放入 a[2]

结果：

下标:  0    1    2    3    4    5
元素: [10] [20] [25] [30] [40] [50]

共移动了 $n - i = 5 - 2 = 3$ 个元素。

删除操作

定理顺序表删除的平均时间复杂度

删除长度为 $n$ 的顺序表中第 $i$ 个元素（$1 \le i \le n$，即下标 $i-1$），需要将后面的 $n - i$ 个元素依次前移一位。

若删除每个位置的概率相同（均为 $\frac{1}{n}$），则平均移动次数为：

$$E = \sum_{i=1}^{n} \frac{1}{n} \cdot (n - i) = \frac{1}{n} \cdot \frac{n(n-1)}{2} \cdot \frac{1}{1} = \frac{n-1}{2}$$

平均时间复杂度仍为 $O(n)$。

例题删除顺序表第 2 个元素（即下标 1 的元素）

设表为 [10, 20, 30, 40, 50]，$n = 5$，删除第 $i=2$ 个元素（值为 20）。

需要移动的元素个数 = $n - i = 5 - 2 = 3$ 个（即 30, 40, 50 分别前移）。

删除前: [10] [20] [30] [40] [50]
删除后: [10] [30] [40] [50]

顺序表的优缺点

优点缺点
支持随机访问，按下标取元素 $O(1)$插入和删除需要移动大量元素，平均 $O(n)$
存储密度高（不需要额外指针）需要预先分配连续空间，可能造成浪费或不够用
缓存友好（Cache Friendly），访问速度快扩容时需要重新分配并复制所有元素
可直接进行二分查找（O(log n)）不适合频繁增删的场景

1.3链表

单链表 (Singly Linked List)

定义单链表

单链表中的每个元素称为一个结点（Node），由两部分组成：

  • 数据域（data）：存储元素值
  • 指针域（next）：存储下一个结点的引用（地址）

最后一个结点的 next 指向 None（空），表示链表结束。

通常用一个变量 head 引用链表的第一个结点（头结点/首结点）。

直觉链表的形象比喻

想象一场寻宝游戏：每个藏宝点有两样东西——一份"宝物"（data）和一张"下一个线索的地图"（next）。你必须从第一个线索开始，一步步找到下一个，不能直接跳到中间——这就是为什么链表不支持随机访问。

单链表的结构示意图：

head → [data|next] → [data|next] → [data|next] → None
        结点1          结点2          结点3

单链表的核心操作

插入操作（在结点 p 后面插入新结点 s）：

  1. 创建新结点 s，设置 s.data = 新数据
  2. 让 s.next 指向 p.next（先连后面）
  3. 让 p.next 指向 s（再连前面）
# 在 p 后面插入 s
s.next = p.next   # 新结点先指向 p 的后继
p.next = s        # p 指向新结点
易错指针操作的顺序不能颠倒！

如果先执行 p.next = s，那么 p 原来的后继结点的地址就丢失了，无法再让 s.next 指向它。所以必须先让新结点指向后面，再修改前面的指针。

删除操作（删除结点 p 的后继结点）：

  1. 用临时变量 q 保存要删除的结点：q = p.next
  2. 让 p.next 跳过 q，指向 q.next：p.next = q.next
  3. （Python 中无需手动释放内存，垃圾回收会处理 q）
# 删除 p 后面的结点
q = p.next        # q 是要删除的结点
p.next = q.next   # 跳过 q

双向链表 (Doubly Linked List)

定义双向链表

双向链表的每个结点有三个部分：

  • prev：指向前驱结点
  • data：数据域
  • next：指向后继结点

这样就可以从任意结点双向遍历，既能向前也能向后。

None ← [prev|data|next] ⇄ [prev|data|next] ⇄ [prev|data|next] → None
           结点1              结点2              结点3

双向链表在 p 结点前插入 q 的操作

这是考试高频考点！正确的步骤：

  1. 设置 q 的前驱：q.prev = p.prev
  2. 设置 q 的后继：q.next = p
  3. 让 p 原来的前驱指向 q：p.prev.next = q
  4. 让 p 的前驱变为 q：p.prev = q
易错修改顺序的陷阱

关键原则：先修改新结点的指针，再修改旧结点的指针。特别注意 p.prev = q 必须在 p.prev.next = q 之后，因为一旦执行了 p.prev = q，就无法再通过 p.prev 访问到 p 的原前驱结点了。

合法的顺序组合有多种，但 2022 年考题的正确答案是：p.prev.next=q; q.next=p; q.prev=p.prev; p.prev=q;（选项 D 的顺序）。

循环链表 (Circular Linked List)

定义循环链表

循环链表是一种特殊的链表，其尾结点的 next 指针不指向 None，而是指向头结点（或链表的第一个结点），形成一个闭环。

  • 循环单链表：尾结点的 next → 头结点
  • 循环双向链表：尾结点的 next → 头结点，头结点的 prev → 尾结点
循环单链表示意图：
head → [A|→] → [B|→] → [C|→] ─┐
  ↑                              │
  └──────────────────────────────┘
直觉何时使用循环链表？

当你需要反复循环遍历一组元素时（如约瑟夫问题、轮询调度），循环链表非常自然——遍历到"尾部"后会自动回到"头部"，不需要特殊处理边界。

另外，如果用尾指针（tail）而非头指针来引用循环链表，可以 $O(1)$ 访问表头和表尾：

  • 表尾：tail
  • 表头：tail.next

链表操作的时间复杂度

操作时间复杂度说明
按下标访问第 $i$ 个元素$O(n)$必须从头遍历到第 $i$ 个
按值查找$O(n)$最坏需要遍历整个链表
在已知结点后插入$O(1)$只需修改指针
在已知结点后删除$O(1)$只需修改指针
在头部插入/删除$O(1)$直接操作 head
在尾部插入$O(n)$ 或 $O(1)$无尾指针时需遍历到尾；有尾指针则 $O(1)$
易错链表的删除/查找时间复杂度

"链表删除是 $O(1)$"——这句话不完全正确！

  • 如果已经知道要删除结点的前驱结点的位置，修改指针确实是 $O(1)$。
  • 但如果只知道要删除的值，则需要先查找该元素，查找本身是 $O(n)$。

因此，"在链表中查找并删除一个元素"的总时间复杂度是 $O(n)$，而不是 $O(1)$。（2024 年判断题考点）

链表的优点

  • 不需要预先分配空间：按需动态创建结点
  • 插入和删除不需要移动元素：只需修改指针，$O(1)$
  • 不会浪费空间：有多少元素就占多少空间（加上指针的额外开销）

链表的缺点

  • 不支持随机访问：必须从头遍历，$O(n)$
  • 额外的指针开销：每个结点除了数据还要存储指针
  • 缓存不友好：结点在内存中不连续，CPU 缓存命中率低
  • 无法二分查找：因为不能随机访问中间元素

1.4顺序表 vs 链表的选择

顺序表与链表的全面对比
比较维度顺序表（数组）链表
存储方式连续内存离散结点 + 指针
随机访问$O(1)$ ✓$O(n)$ ✗
头部插入/删除$O(n)$（需移动所有元素）$O(1)$ ✓
中间插入/删除$O(n)$查找 $O(n)$ + 修改 $O(1)$
尾部插入均摊 $O(1)$（动态数组）$O(n)$（无尾指针）/ $O(1)$（有尾指针）
空间分配预分配，可能浪费按需分配，无浪费
二分查找✓ 可以✗ 不可以
缓存性能好（连续内存）差（离散存储）
额外空间开销无每结点一个/两个指针
选择策略：频繁随机访问 → 顺序表；频繁在头部或中间插入删除 → 链表；需要二分查找 → 必须用顺序存储。
直觉考试常见的场景判断题
  • "经常需要在线性表头部进行插入和删除" → 最合适的是链表（$O(1)$）
  • "需要随机存取" → 最合适的是顺序表（$O(1)$ 按下标访问）
  • "有序表上做二分查找" → 必须用顺序存储
  • "元素个数变化很大、事先不知道规模" → 链表更好
  • "删除表中第 1 个元素" → 链表效率更高（$O(1)$ vs 顺序表 $O(n)$）

1.5Python 中链表的实现

定义Node 类 — 链表结点
class Node:
    def __init__(self, data, next=None):
        self.data = data  # 数据域
        self.next = next  # 指针域，默认为 None

一个 Node 对象就是链表中的一个结点。data 存放数据，next 存放对下一个 Node 的引用。

创建链表

# 手动创建一个包含 1→2→3 的链表
node3 = Node(3)          # 尾结点，next 默认为 None
node2 = Node(2, node3)   # 中间结点，next 指向 node3
node1 = Node(1, node2)   # 头结点，next 指向 node2
head = node1             # head 引用链表的第一个结点

遍历链表

def print_list(head):
    p = head
    while p is not None:
        print(p.data, end=" ")
        p = p.next
    print()

从列表构建链表

class LinkList:
    def __init__(self, lst):
        self.head = Node(lst[0])
        p = self.head
        for i in lst[1:]:
            node = Node(i)
            p.next = node   # 当前尾结点指向新结点
            p = node        # p 移动到新的尾结点

    def print(self):
        p = self.head
        while p:
            print(p.data, end=" ")
            p = p.next
        print()

链表反转（头插法）

例题单链表原地反转

思路：遍历链表，将每个结点依次插入到链表头部。

def reverse(self):
    p = self.head.next    # p 指向第二个结点
    self.head.next = None # 头结点变成尾结点
    while p is not None:
        q = p             # 保存当前结点
        p = p.next        # p 移动到下一个
        q.next = self.head.next  # q 插入到 head 后面（非 head 本身！）
        self.head.next = q       # 错误理解：实际是 q 插到 head 之后
    # 注意：这种写法是把 head 作为"哨兵"留在最前面
    # 另一种常见写法（head 也参与反转）见下方

另一种更清晰的写法（直接改 head）：

def reverse(self):
    prev = None
    curr = self.head
    while curr is not None:
        next_node = curr.next  # 保存下一个
        curr.next = prev       # 反转指针
        prev = curr            # prev 前进
        curr = next_node       # curr 前进
    self.head = prev           # 新的头是原来的尾

真题选择题精选

选择链表的特点辨析
2021 期末 选择题 链表特性 ★☆☆
链表不具有的特点是（　）。
A. 可随机访问任意元素
B. 插入和删除不需要移动元素
C. 不必事先估计存储空间
D. 所需空间与线性表长度成正比
A

解析：

  • A. 可随机访问任意元素——错误！链表不能随机访问。要访问第 $i$ 个元素必须从头遍历，时间 $O(n)$。随机访问是顺序表的特点。
  • B. 插入和删除不需要移动元素——正确。链表只需修改指针。
  • C. 不必事先估计存储空间——正确。链表按需动态分配。
  • D. 所需空间与线性表长度成正比——正确。每个结点占固定空间，$n$ 个结点共占 $\Theta(n)$ 空间。

所以链表不具有的特点是 A。

选择链表判空条件
2021 期末 选择题 链表基本操作 ★☆☆
判定一个无序表 Q（链表实现）为空的条件是（　）。
A. Q.head == None
B. Q == None
C. Q.head == 0
D. Q.head != None
A

解析：

  • 链表为空意味着没有任何结点，即头指针 head 不指向任何结点。在 Python 中用 None 表示空引用。
  • B 错误：Q 是链表对象本身，只要被创建就不是 None；即使链表为空，Q 对象仍然存在。
  • C 错误：Python 中空引用用 None，不是 0。
  • D 错误：Q.head != None 表示链表非空，逻辑完全反了。
选择双向链表插入操作
2022 期末 选择题 双向链表指针操作 ★★★ 高频考点
双向链表中的每个结点有两个引用域，prev 和 next，分别引用当前结点的前驱与后继，设 p 引用链表中的一个结点，q 引用一待插入结点，现要求在 p 前插入 q，则正确的插入操作为（　）。
A. p.prev=q; q.next=p; p.prev.next=q; q.prev=p.prev;
B. q.prev=p.prev; p.prev.next=q; q.next=p; p.prev=q.next;
C. q.next=p; p.next=q; p.prev.next=q; q.next=p;
D. p.prev.next=q; q.next=p; q.prev=p.prev; p.prev=q;
D

解析：在 p 前插入 q，需要修改四个指针：

  1. p 的原前驱的 next 指向 q：p.prev.next = q（此时 p.prev 还指向原前驱，所以可以正确找到）
  2. q 的 next 指向 p：q.next = p
  3. q 的 prev 指向 p 的原前驱：q.prev = p.prev（此时 p.prev 还没被改，仍是原前驱）
  4. p 的 prev 指向 q：p.prev = q（最后才改 p.prev，因为前面还需要用它）

逐一验证选项 D：p.prev.next=q; q.next=p; q.prev=p.prev; p.prev=q;

  • 第 1 步：p.prev.next = q → 原前驱的 next 正确指向 q ✓
  • 第 2 步：q.next = p → q 后面是 p ✓
  • 第 3 步：q.prev = p.prev → 此时 p.prev 还未被修改，仍是原前驱 ✓
  • 第 4 步：p.prev = q → 最后修改 p.prev ✓

选项 A 第一步就把 p.prev 改了，导致后续无法找到原前驱。选项 B 最后一步 p.prev=q.next 等于 p.prev=p，荒谬。选项 C 中 p.next=q 修改了 p 的后继，完全错误。

解析：

  • 线性表——是一种逻辑结构（描述元素间的一对一关系）
  • 链表——是一种存储结构（描述如何在内存中用指针链接结点来存储线性表）

选择顺序表 vs 链表效率比较
2024 期末 选择题 顺序表与链表比较 ★★☆
假设线性表有 2n（n > 100）个元素，以下操作（　）在单链表上实现比在顺序表上实现效率更高。
A. 在表中最后一个元素的后面插入一个新元素
B. 顺序输出表中的前 i 个元素
C. 交换表中第 i 个元素和第 n+i 个元素的值 (i=0, ..., n-1)
D. 删除表中第 1 个元素
D

逐项分析：

  • A. 在最后一个元素后插入：顺序表均摊 $O(1)$（尾部追加）；链表若无尾指针需 $O(n)$ 遍历到末尾。顺序表更优。
  • B. 顺序输出前 i 个元素：两者都是 $O(i)$，顺序表因为缓存友好可能更快。没有优势。
  • C. 交换第 i 和第 n+i 个元素：顺序表 $O(1)$ 按下标直接访问；链表每次要遍历 $O(n)$。顺序表远优。
  • D. 删除第 1 个元素：链表 $O(1)$（直接修改 head）；顺序表 $O(n)$（需要把后面所有元素前移）。链表更优！

填空二分检索的存储要求

顺序

解析：二分查找（Binary Search）每次需要访问中间位置的元素。这要求能在 $O(1)$ 时间内通过下标访问任意位置——只有顺序存储（数组）能做到。链表无法在 $O(1)$ 内访问中间元素，因此无法高效地进行二分查找。

填空完全二叉树的顺序存储编号
2021 期末 填空题 顺序存储 + 完全二叉树 ★★☆
对于具有 57 个结点的完全二叉树，如果按层次自顶向下，同一层自左向右，顺序从 0 开始对全部结点进行编号，则有：编号为 18 的结点的父结点的编号是___，编号为 19 的结点的右子女结点的编号是___。
8；40

解析：完全二叉树按层序从 0 编号的性质（这是顺序存储的基础）：

  • 结点 $i$ 的父结点编号 = $\lfloor(i-1)/2\rfloor$
  • 结点 $i$ 的左子女编号 = $2i + 1$
  • 结点 $i$ 的右子女编号 = $2i + 2$

计算：

  • 编号 18 的父结点 = $\lfloor(18-1)/2\rfloor = \lfloor 17/2 \rfloor = 8$
  • 编号 19 的右子女 = $2 \times 19 + 2 = 40$（需验证 $40 < 57$，确实存在）
填空顺序表删除移动元素数
2022 期末 填空题 顺序表删除 ★☆☆
删除长度为 n 的顺序表的第 i 个数据元素需要移动表中的____个数据元素。（$1 \le i \le n$）
$n - i$

解析：删除第 $i$ 个元素后，其后面的第 $i+1, i+2, \ldots, n$ 个元素都要向前移一位，共 $n - i$ 个。

验证边界：

  • $i = 1$（删除第一个）：移动 $n - 1$ 个 ✓
  • $i = n$（删除最后一个）：移动 $0$ 个 ✓
填空存储结构选择
2024 期末 填空题 顺序表与链表选择 ★☆☆
按照简单且高效的原则，如果经常需要在线性表头部进行插入和删除操作，最合适的存储结构是____；如果需要随机存取，最合适的存储结构是____。
链表；顺序表

解析：

  • 头部插入/删除：链表只需修改 head 指针，$O(1)$；顺序表需要移动所有元素，$O(n)$。所以选链表。

思路：尾插法。变量 p 始终指向当前链表的最后一个结点。

  2. ② node：p = node —— 更新尾指针，让 p 指向新的尾结点（刚创建的 node）。

  1. ③ None：self.head.next = None —— head 原来的后继断开（head 本身保留在最前面作为第一个数据结点，它将变成反转后的最后一个结点，所以其 next 设为 None）。
  2. ④ p.next：p = p.next —— 在 q "摘走" 当前结点之前，p 先前进到下一个结点，否则链会断。
  3. ⑤ self.head.next：q.next = self.head.next —— 让摘下的结点 q 指向 head 的当前后继（即已反转部分的第二个结点），准备将 q 插入到 head 后面。
  4. ⑥ self.head.next = q —— 完成头插：让 head 的 next 指向 q。

执行过程模拟

设原始链表为 head → 1 → 2 → 3 → None：

初始: head(1) → 2 → 3 → None
③后: head(1) → None, p → 2 → 3 → None

第1轮: q=2, p=3, q.next=None(即head.next), head.next=q
       → head(1) → 2 → None, p → 3

第2轮: q=3, p=None, q.next=2(即head.next), head.next=q
       → head(1) → 3 → 2 → None

循环链表结构理解

带尾指针的循环单链表：

     ┌───────────────────────────────┐
     ↓                               │
    [A] → [B] → [C] → ... → [tail] ─┘
     ↑                         ↑
 tail.next                   self.tail
  • 表头（第一个结点）= self.tail.next
  • 表尾（最后一个结点）= self.tail
  • 尾结点的 next 指回头结点（形成环）

逐空详解

(1) nd.next = self.tail.next

pushFront（头插）：新结点要成为新的表头。在循环链表中，表头 = tail.next。所以新结点的 next 应指向当前表头（即 self.tail.next）。紧接着下一行 self.tail.next = nd 让尾结点指向新头。

插入前: tail → [原头] → ... → [tail] ─回→ [原头]
插入后: tail → [nd(新头)] → [原头] → ... → [tail] ─回→ [nd]

(2) self.tail.next is self.tail

popBack（尾删）：如果链表只有一个结点，该结点的 next 指向自己（tail.next == tail）。此时删除后链表为空，直接设 self.tail = None。

(3) ptr.next != self.tail

要删除尾结点，需要找到尾结点的前驱。从头开始遍历，当 ptr.next == self.tail 时停下——此时 ptr 就是尾结点的前驱。所以循环条件是"ptr 的下一个还不是 tail"。

(4) self.tail = ptr

删除尾结点后，它的前驱 ptr 成为新的尾结点。上一行 ptr.next = ptr.next.next 已经让 ptr 跳过了原 tail 指向表头（维持循环），现在只需更新 tail 指针。

(5) ptr != self.tail.next

print 遍历：从 tail.next（表头）开始，绕一圈回到表头时停止。由于第一个元素在 while 之前已经打印了，所以 while 条件检测的是"还没回到起点"。当 ptr 再次等于 self.tail.next 时说明绕了一圈，应停止。
