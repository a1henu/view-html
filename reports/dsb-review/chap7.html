第七章 算法设计基础

分治法、动态规划与回溯法是三种核心的算法设计策略。本章系统讲解这三种方法的基本思想、适用条件、典型应用，并对比它们的异同，帮助在考试中快速判断应使用哪种策略。

7.1算法复杂度分析基础

在讨论具体的算法设计策略之前，我们首先回顾复杂度分析的基本工具。复杂度分析是评估算法性能的基石，也是理解分治递推方程的前提。

渐进记号

定义大 O 记号（上界）

$f(n) = O(g(n))$ 表示存在正常数 $c$ 和 $n_0$，使得对所有 $n \geq n_0$，有：

$$f(n) \leq c \cdot g(n)$$

直觉：$O(g(n))$ 给出了 $f(n)$ 增长速度的上界——"最坏情况下也不会比这更慢"。

定义大 $\Omega$ 记号（下界）

$f(n) = \Omega(g(n))$ 表示存在正常数 $c$ 和 $n_0$，使得对所有 $n \geq n_0$，有：

$$f(n) \geq c \cdot g(n)$$

直觉：$\Omega(g(n))$ 给出了增长速度的下界——"至少需要这么多时间"。

定义大 $\Theta$ 记号（精确界）

$f(n) = \Theta(g(n))$ 当且仅当 $f(n) = O(g(n))$ 且 $f(n) = \Omega(g(n))$。

直觉：$\Theta$ 同时限定了上界和下界，表示 $f(n)$ 与 $g(n)$ 具有相同量级的增长率。

常见复杂度排序

从低到高排列：$O(1) \lt O(\log n) \lt O(n) \lt O(n \log n) \lt O(n^2) \lt O(n^3) \lt O(2^n) \lt O(n!)$
常见复杂度与典型算法
复杂度名称典型算法
$O(1)$常数时间数组随机访问、哈希查找
$O(\log n)$对数时间二分查找、平衡BST查找
$O(n)$线性时间线性搜索、遍历链表
$O(n \log n)$线性对数时间归并排序、快速排序（期望）
$O(n^2)$平方时间冒泡排序、选择排序
$O(2^n)$指数时间穷举子集、朴素递归斐波那契
易错什么不影响时间复杂度？

时间复杂度取决于问题的规模 $n$ 和输入值的分布（如最坏/平均/最好情况），而不取决于计算结果。

例如：一个函数计算 $1 + 2 + \cdots + n$，无论结果值 $n(n+1)/2$ 多大，只要用公式直接计算就是 $O(1)$；用循环累加就是 $O(n)$——复杂度由算法策略和输入规模决定，和输出的数值无关。

递推关系与主定理

分治算法的时间复杂度通常用递推关系（recurrence relation）描述。掌握几种典型递推的求解方法是本章重点。

定理主定理（Master Theorem）

设递推关系为：

$$T(n) = a \cdot T\!\left(\frac{n}{b}\right) + O(n^d)$$

其中 $a \geq 1$，$b > 1$，$d \geq 0$。令 $c = \log_b a$，则：

$$T(n) = \begin{cases} O(n^d) & \text{若 } d > c \ (\text{即 } a < b^d) \\ O(n^d \log n) & \text{若 } d = c \ (\text{即 } a = b^d) \\ O(n^c) = O(n^{\log_b a}) & \text{若 } d < c \ (\text{即 } a > b^d) \end{cases}$$

直觉：比较"合并代价的增长指数 $d$"与"子问题增长指数 $\log_b a$"——谁更大谁主导总复杂度；若打平则多乘一个 $\log$。

例题用主定理分析归并排序

递推：$T(n) = 2T(n/2) + O(n)$

对应参数：$a = 2$，$b = 2$，$d = 1$

  1. 计算 $c = \log_b a = \log_2 2 = 1$
  2. 比较 $d$ 与 $c$：$d = 1 = c$，落入"打平"情况
  3. 结论：$T(n) = O(n^1 \cdot \log n) = O(n \log n)$
例题用主定理分析二分查找

递推：$T(n) = T(n/2) + O(1)$

对应参数：$a = 1$，$b = 2$，$d = 0$

  1. 计算 $c = \log_b a = \log_2 1 = 0$
  2. 比较 $d$ 与 $c$：$d = 0 = c$，落入"打平"情况
  3. 结论：$T(n) = O(n^0 \cdot \log n) = O(\log n)$
注记更多递推示例
  • $T(n) = 4T(n/2) + O(n)$：$a=4, b=2, d=1$，$c = \log_2 4 = 2 > d$，故 $T(n) = O(n^2)$
  • $T(n) = 2T(n/2) + O(n^2)$：$a=2, b=2, d=2$，$c = 1 < d$，故 $T(n) = O(n^2)$
  • $T(n) = 3T(n/3) + O(n)$：$a=3, b=3, d=1$，$c = 1 = d$，故 $T(n) = O(n \log n)$

7.2分治法（Divide and Conquer）

基本思想与三个步骤

定义分治法 (Divide and Conquer)

将一个规模为 $n$ 的问题分解为若干个规模较小但类似于原问题的子问题，递归地求解这些子问题，然后将子问题的解合并为原问题的解。

直觉三步走：分-治-合

分治法的核心可以概括为三个步骤：

  1. 分解（Divide）：将原问题划分为若干规模更小的同类子问题。
  2. 解决（Conquer）：递归地解决各子问题。当子问题规模足够小时，直接求解（递归出口）。
  3. 合并（Combine）：将子问题的解组合/合并成原问题的解。

注意：分治的精髓在于"合并"步骤——如果子问题的解不需要合并，那可能只需要简单递归即可（如二分查找只取一个子问题的结果）。

适用条件

定义分治法的四个适用条件
  1. 规模缩小可解：问题规模缩小到一定程度后容易直接解决
  2. 可分解性：问题可以分解为若干个规模较小的同类问题
  3. 解可合并：子问题的解可以合并为原问题的解
  4. 子问题相互独立：各子问题之间没有重叠（互不依赖）

第4点尤为关键——如果子问题有重叠（同一个子问题被反复求解），则分治会导致大量重复计算，此时应该转用动态规划。

典型应用

归并排序 (Merge Sort)

例题归并排序的分治过程

问题：将一个无序数组 $A[0..n-1]$ 排序。

  1. 分解：将数组从中间一分为二 → $A[0..m-1]$ 和 $A[m..n-1]$，其中 $m = \lfloor n/2 \rfloor$
  2. 解决：递归地对左半和右半分别排序
  3. 合并：将两个已排序子数组合并为一个有序数组（双指针归并，$O(n)$）

递推方程：$T(n) = 2T(n/2) + O(n)$

时间复杂度：$O(n \log n)$（各情况均为 $\Theta(n\log n)$）

空间复杂度：$O(n)$（需要辅助数组）

快速排序 (Quick Sort)

例题快速排序的分治过程
  1. 分解：选择一个 pivot 元素，将数组划分为"小于 pivot"和"大于 pivot"两部分
  2. 解决：递归地对两个子数组分别排序
  3. 合并：无需额外合并（分区后自然有序）

时间复杂度：

  • 最坏情况：$O(n^2)$（每次选到最大或最小元素作 pivot）
  • 期望（平均）情况：$O(n \log n)$

二分查找 (Binary Search)

例题二分查找的分治过程

前提：数组已排序。

  1. 分解：将查找区间一分为二
  2. 解决：比较中间元素与目标值，确定目标在左半还是右半，递归进入相应子区间
  3. 合并：无需合并（只取一个子问题的结果）

递推方程：$T(n) = T(n/2) + O(1)$

时间复杂度：$O(\log n)$

注记分治的通用递推框架

设分治算法将问题分为 $a$ 个规模为 $n/b$ 的子问题，合并的代价为 $f(n)$，则：

$$T(n) = a \cdot T(n/b) + f(n)$$

这正是主定理的形式。不同的 $a, b, f(n)$ 组合对应不同的算法效率。

7.3动态规划（Dynamic Programming）

基本思想

定义动态规划 (Dynamic Programming, DP)

将问题分解为重叠的子问题，用一张表（数组）保存已经计算过的子问题的解，避免重复计算，自底向上（bottom-up）逐步构建出原问题的最优解。

直觉动态规划 = 记忆化 + 递推

如果你发现递归解法中有大量重复计算（如斐波那契 $F(n) = F(n-1) + F(n-2)$，$F(3)$ 被算了无数遍），那就把算过的结果存起来：

  • 自顶向下 + 记忆化（Memoization）：仍然递归，但遇到已算过的子问题直接查表
  • 自底向上 + 表格（Tabulation）：按顺序从最小的子问题开始算，逐步填表到原问题

两种方式等价，但自底向上通常更省递归栈空间。

适用条件（两个关键性质）

定义最优子结构 (Optimal Substructure)

问题的最优解包含其子问题的最优解。即：如果我们已经知道子问题的最优解，就能构造出原问题的最优解。

举例：最短路径问题——从 A 到 C 的最短路径经过 B，则 A→B 的那段一定是 A 到 B 的最短路径。

定义重叠子问题 (Overlapping Subproblems)

递归求解时，同一个子问题会被反复求解多次（子问题之间并不独立）。

举例：计算 $F(5) = F(4) + F(3)$，而 $F(4) = F(3) + F(2)$——$F(3)$ 被重复计算。

易错如何判断是否适用动态规划？

必须判定问题是否具有最优子结构性质。仅有重叠子问题而无最优子结构，DP 也无法给出正确的最优解。

判断流程：

  1. 能否把问题定义为"在某种约束下求最值"的形式？
  2. 原问题的最优解能否由子问题的最优解推出（即"剪切粘贴"论证能否成立）？
  3. 子问题之间是否有重叠？

三者皆具备 → 适用 DP。

经典例子

斐波那契数列

例题斐波那契数列的 DP 解法

问题：$F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2)$

朴素递归：$T(n) = O(2^n)$（大量重复计算）

DP（自底向上）：

  1. 建表 $dp[0..n]$，初始化 $dp[0] = 0, dp[1] = 1$
  2. 依次计算 $dp[i] = dp[i-1] + dp[i-2]$，$i = 2, 3, \ldots, n$
  3. 最终答案为 $dp[n]$

时间复杂度：$O(n)$（每个子问题只算一次）

空间复杂度：$O(n)$（可优化为 $O(1)$，只保留前两项）

0-1 背包问题

例题0-1 背包问题

问题：有 $n$ 个物品，第 $i$ 个物品重量为 $w_i$、价值为 $v_i$。背包容量为 $W$。每个物品只能选或不选，求装入背包的物品的最大总价值。

状态定义：$dp[i][j]$ = 前 $i$ 个物品中选取、总重量不超过 $j$ 时的最大价值。

状态转移方程：

$$dp[i][j] = \begin{cases} dp[i-1][j] & \text{若 } w_i > j \ (\text{放不下}) \\ \max(dp[i-1][j], \ dp[i-1][j-w_i] + v_i) & \text{若 } w_i \leq j \end{cases}$$
  1. 不选第 $i$ 个物品：最大价值等于前 $i-1$ 个物品在容量 $j$ 下的最大价值 $dp[i-1][j]$
  2. 选第 $i$ 个物品：价值 = 前 $i-1$ 个物品在容量 $j - w_i$ 下的最大价值 + $v_i$
  3. 取两者的 $\max$ 即为 $dp[i][j]$ 的值

时间复杂度：$O(nW)$

最优子结构：如果第 $i$ 个物品在最优解中，则去掉它后，剩余物品在容量 $W-w_i$ 中仍是最优的。

最短路径

例题Floyd 全源最短路径

状态：$dp[k][i][j]$ = 从 $i$ 到 $j$，只允许经过编号 $\leq k$ 的中间结点时的最短路径长度。

转移：$dp[k][i][j] = \min(dp[k-1][i][j], \ dp[k-1][i][k] + dp[k-1][k][j])$

时间复杂度：$O(n^3)$

Floyd 算法体现了 DP 的"逐步放开约束"思想——每轮新增一个可中转的结点。

7.4回溯法（Backtracking）

基本思想与特点

定义回溯法 (Backtracking)

回溯法是一种系统地搜索问题的所有解或任意一个解的算法。它采用深度优先搜索（DFS）策略遍历解空间树，在搜索过程中：

  • 在生成解空间的任一结点时，先判断当前结点是否可能包含问题的有效解；
  • 如果肯定不包含有效解，则跳过以该结点为根的整棵子树的搜索（剪枝），逐层向祖先结点回溯；
  • 如果可能包含，则继续向下扩展。
直觉回溯 = DFS + 剪枝

你可以把回溯法想象成"走迷宫"：

  • 每到一个岔路口，选一条路走下去（DFS 展开）
  • 如果走到死胡同（约束不满足），就退回上一个岔路口，换一条路试（回溯）
  • 如果能提前判断某条路一定走不通，就根本不进去（剪枝）

回溯法既有系统性（不遗漏任何可能的解）又有跳跃性（通过剪枝跳过无效分支）。

定义回溯法的关键特点
  1. 可以系统地搜索一个问题的所有解或任意解
  2. 是一种既具备系统性又具备跳跃性的搜索算法
  3. 需要借助栈（或递归调用栈）来保存从根结点到当前扩展结点的路径
  4. 搜索方式：深度优先搜索解空间树
  5. 核心优化：剪枝函数（约束函数 + 界限函数）
易错回溯法用"栈"而非"队列"

回溯法基于深度优先搜索（DFS），对应的数据结构是栈（或等价地，递归调用的系统栈）。

如果用队列辅助搜索，那是广度优先搜索（BFS），不是回溯法。这是考试中非常容易混淆的知识点。

记忆：回溯 = 回头（后退） = 后进先出 = 栈。

典型应用

N 皇后问题

例题N 皇后问题

问题：在 $N \times N$ 的棋盘上放置 $N$ 个皇后，使得任意两个皇后不在同一行、同一列、同一对角线。

解空间：用 $x[i]$ 表示第 $i$ 行皇后所在的列号（$1 \leq x[i] \leq N$）

约束条件：

  • 不同列：$x[i] \neq x[j]$（$\forall i \neq j$）
  • 不同对角线：$|x[i] - x[j]| \neq |i - j|$

回溯过程：

  1. 逐行放置：第 1 行的皇后尝试放在第 1 到第 N 列
  2. 每放一个，检查约束：是否和已放置的皇后冲突
  3. 若不冲突，进入下一行继续放置（递归/DFS）
  4. 若冲突，换同一行的下一列尝试；若该行所有列都不行，则回溯到上一行
  5. 成功放置 $N$ 个皇后后，输出一个解；如需所有解，继续回溯寻找其他方案

4 皇后的一个解：$x = [2, 4, 1, 3]$（即第1行第2列、第2行第4列、第3行第1列、第4行第3列）。

迷宫问题

例题迷宫求解

问题：从迷宫入口找一条通往出口的路径。

回溯思路：从入口开始，尝试上下左右四个方向走：

  • 走得通且未走过 → 标记已走，继续深入
  • 走不通（墙壁或已走过）→ 回退到上一个位置，换方向
  • 到达出口 → 输出路径

这就是典型的 DFS + 回溯。用栈保存当前路径，回退时弹栈。

图的着色问题

例题图的 m 着色问题

问题：用 $m$ 种颜色给图的 $n$ 个结点着色，要求相邻结点颜色不同。

解空间：$x[i] \in \{1, 2, \ldots, m\}$ 表示结点 $i$ 的颜色

约束函数：对于每条边 $(i, j)$，要求 $x[i] \neq x[j]$

过程：逐个结点着色，每次检查是否与已着色的邻接结点冲突，冲突则换色或回溯。

7.5贪心算法（Greedy Algorithm）

定义贪心算法

贪心算法在每一步都做出当前看来局部最优的选择（贪心选择），期望通过局部最优的叠加达到全局最优。

特点：

  • 每步只考虑当前状态，不考虑未来的影响
  • 一旦做出选择就不再回退（与回溯法的区别）
  • 不一定能得到全局最优解——只有当问题具有贪心选择性质和最优子结构时，贪心才能保证最优
注记贪心算法的典型应用
  • Prim 算法：最小生成树——每次选当前最短的跨切割边
  • Kruskal 算法：最小生成树——每次选全局最短的不构成环的边
  • Dijkstra 算法：单源最短路径——每次选距离最小的未访问结点
  • 哈夫曼编码：每次合并频率最小的两个结点
  • 活动选择问题：每次选最早结束的活动

这些算法在前面的章节（图论、树）中已经详细讨论过，此处仅点明它们属于贪心策略。

7.6三种策略对比

分治、动态规划、回溯（加上贪心）是算法设计的四大策略。它们有相似之处（都涉及子问题的分解），但在适用场景和执行方式上有本质区别。

分治法 vs 动态规划 vs 回溯法 vs 贪心
对比维度 分治法 动态规划 回溯法 贪心算法
核心思想 分解为独立子问题，递归求解后合并 分解为重叠子问题，存表避免重复计算 DFS 搜索解空间树 + 剪枝 每步局部最优选择
子问题关系 子问题互相独立 子问题有重叠 解空间中各候选解可能共享前缀 无需考虑子问题（贪心选择后不回头）
求解方向 自顶向下（递归） 通常自底向上（也可自顶向下 + 记忆化） 深度优先（从根到叶） 自顶向下（逐步决策）
是否保证最优 是（求解确定问题） 是（在满足最优子结构时） 是（穷举所有解中的最优） 不一定（需证明贪心选择性质）
时间复杂度 通常 $O(n \log n)$ 取决于状态数，通常多项式 最坏指数级（但剪枝可大幅减少） 通常 $O(n \log n)$ 或更低
空间开销 递归栈 + 合并辅助空间 状态表（$dp$ 数组） 递归栈（路径深度） $O(1)$ 到 $O(n)$
适用场景 子问题独立可合并（排序、查找） 最优化问题，有最优子结构和重叠子问题 组合搜索、约束满足问题 有贪心选择性质的最优化问题
典型算法 归并排序、快排、二分查找 背包、最短路径、最长子序列 N 皇后、迷宫、图着色 Dijkstra、Prim、Kruskal、哈夫曼
直觉一句话区分四种策略
  • 分治："分头干活，各干各的，最后汇总"——子问题独立
  • DP："大家的活有交叉，算过的记下来别重复"——子问题重叠
  • 回溯："挨个试，不行就退回来换一个"——穷举 + 剪枝
  • 贪心："每步都挑眼前最好的，走了不回头"——局部最优
分治 vs 动态规划的核心区别：两者都将问题分解为规模更小的子问题，但分治法的子问题互相独立（无重叠），动态规划的子问题有重叠（需要存表避免重复计算）。

真题往年真题精选

本章在历年考试中主要以选择题和判断题形式考查，重点在于理解三种策略的基本概念和区别。

真题算法时间复杂性的影响因素
2021 选择题 复杂度分析 难度 ★
下列不影响算法时间复杂性的因素有（ ）。
A. 问题的规模　　B. 输入值　　C. 计算结果　　D. 算法的策略
C

解析：

  • A. 问题的规模：影响。$n$ 越大，时间通常越多。
  • B. 输入值：影响。例如快排的最坏情况取决于输入的排列。
  • C. 计算结果：不影响。时间复杂度衡量的是"执行步骤数量与输入规模的关系"，与输出的数值无关。
  • D. 算法的策略：影响。同一问题用不同算法（如冒泡 vs 归并）复杂度不同。
真题回溯法的特性
2022 选择题 回溯法 难度 ★★
回溯法是一类广泛使用的算法，以下叙述中不正确的是（ ）。
A. 回溯法可以系统地搜索一个问题的所有解或者任意解
B. 回溯法是一种既具备系统性又具跳跃性的搜索算法
C. 回溯法需要借助队列数据结构来保存从根结点到当前扩展结点的路径
D. 回溯法在生成解空间的任一结点时，先判断当前结点是否可能包含问题的有效解，如果肯定不包含，则跳过对该结点为根的子树的搜索，逐层向祖先结点回溯
C

解析：

  • A 正确：回溯法的目标就是系统搜索所有解或任意一个解。
  • B 正确：系统性——不遗漏；跳跃性——通过剪枝跳过无效子树。
  • C 不正确：回溯法基于深度优先搜索，使用的是栈（或递归的调用栈），不是队列。队列对应的是广度优先搜索（BFS）。
  • D 正确：这正是回溯法的剪枝机制的准确描述。

记忆口诀：回溯 → 回退 → 后进先出 → 栈；广搜 → 层层扩展 → 先进先出 → 队列。

真题分治算法的递归特性
2022 判断题 分治法 难度 ★
分治算法通常将原问题分解为几个规模较小但类似于原问题的子问题，并要求算法实现写成某种递归形式，递归地求解这些子问题，然后再合并这些子问题的解来建立原问题的解。（Y / N）
Y（正确）

解析：这是分治法的标准定义。分治法的三个步骤：分解（划分为规模更小的同类子问题）→ 解决（递归求解）→ 合并（把子问题的解组合为原问题的解）。题目描述完全准确。

真题动态规划的判定条件
2022 判断题 动态规划 难度 ★
考察某个具体问题是否适合应用动态规划算法，必须判定它是否具有最优子结构性质。（Y / N）
Y（正确）

解析：动态规划的两个必要条件是最优子结构和重叠子问题。其中，最优子结构是判定 DP 能否正确工作（保证最优性）的关键前提——如果原问题的最优解不能由子问题的最优解构成，DP 就无法得到正确的最优解。因此"必须判定是否具有最优子结构性质"是正确的。

真题分治算法复杂度计算
2024 选择题 分治法 · 主定理 难度 ★★
若使用分治算法解决某一问题，每次把规模为 $n$ 的问题分解为 2 个规模为 $n/2$ 的子问题，将两个子问题的结果合并的时间开销为 $O(n)$，那么这个算法的总时间复杂度为（ ）。
A. $O(n)$　　B. $O(n \log n)$　　C. $O(n^2)$　　D. $O(n^2 \log n)$
B

解析：

  1. 写出递推方程：$T(n) = 2T(n/2) + O(n)$
  2. 对应主定理参数：$a = 2, b = 2, d = 1$
  3. 计算 $\log_b a = \log_2 2 = 1 = d$
  4. 属于"打平"情况：$T(n) = O(n^d \log n) = O(n \log n)$

这与归并排序的复杂度完全一致——实际上归并排序就是这种分治模式的经典实例。

真题分治法与动态规划的共同点
2024 判断题 分治法 vs 动态规划 难度 ★
分治法和动态规划法都运用了将问题分解为规模较小的子问题的思想。（Y / N）
Y（正确）

解析：这是分治和动态规划的共同点——两者都将原问题拆解为规模更小的子问题来求解。

它们的区别在于：

  • 分治法的子问题互相独立（不重叠），每个子问题只算一次
  • 动态规划的子问题有重叠（同一子问题被多次引用），需要用表存储避免重复计算

但"分解为子问题"这个大方向是共有的，所以题目说法正确。
